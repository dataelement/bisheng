# E2E 验证清单: F028 工作台会话导出 & 导入知识空间

**测试环境**：
- Client：http://localhost:4001/workspace
- Backend：http://localhost:7860 (默认 admin 密码 `Bisheng@top1`)

**前置条件**：
- 工作台内已有至少 1 个含多轮 query/answer 的日常模式会话
- 工作台内已有至少 1 个工作流 / 助手会话（`/app/<cid>/<fid>/<type>` 路由）
- 当前账号在至少 1 个知识空间有 `upload_file` 权限
- 准备 1 条曾「重新生成」过的回答（验证多 answer 联选）

---

## Platform / Client 前端

### Selection mode 入口（AC-01）
- [ ] 工作台日常模式：每条 AI 回答下方的 icon 行有 ↓ Download 图标
- [ ] 工作流 / 助手会话页（`/app/...`）：MessageBs 的操作行有 ↓ icon
- [ ] 点击 ↓：进入选择态 → 底部固定栏出现 `全选 / 4 个格式 / 导入到知识空间 / ×`
- [ ] 该 answer + **紧挨上方的 query** 两个 checkbox 自动勾上

### 配对联选（AC-03 / AC-06）
- [ ] 点 query 的 checkbox：相邻 assistant answer 一起切换
- [ ] 点 answer 的 checkbox：对应 query 也跟着切换
- [ ] 「重新生成」产生的多 answer：勾任一 → 整组（query + 所有 sibling answer）联动
- [ ] 取消勾选行为对称（cascading uncheck）

### 全选 / 取消全选（AC-04）
- [ ] 工具栏「全选」勾上 → 所有当前消息变选中
- [ ] 上滑加载更多历史消息 → 新加载的消息**也自动选中**
- [ ] **再点「全选」一次 → 清空所有选择**（含手动勾过的）

### 全选以下消息（AC-05）
- [ ] 选择态下右键 / 长按某条消息 → 出现 SelectAllBelowBanner
- [ ] 点 banner → 该锚点及后续所有消息选中
- [ ] 上滑加载更多后，后续新消息也保持选中

### 流式态隐藏（AC-07）
- [ ] 一条 AI 回答正在流式生成（**未完成**）
- [ ] 该消息下方**不显示** Download icon（与复制 icon 一起隐藏）
- [ ] 流式完成后 icon 出现

### 二次点击退出（custom UX）
- [ ] 选择态下再次点击任一 AI 回答的 ↓ icon → 退出选择态（按钮高亮 + title 变 "退出选择"）

### 工作流非纯文本（AC-09）
- [ ] 工作流会话里含 OUTPUT 节点的表单 / 按钮交互
- [ ] 导出文件中：交互组件渲染为 `[交互组件：form|button|...]` 占位文本

### 导入到知识空间 - Modal（AC-16 / AC-17）
- [ ] 选择态下点「导入到知识空间」→ `AddToKnowledgeModal` 弹出
- [ ] Modal 列出的空间列表是当前用户**有 upload_file 权限**的空间（admin 看全部 SPACE 类型）
- [ ] 顶部搜索框输入关键词 → 列表实时过滤
- [ ] 点击空间 → 可展开看到子文件夹
- [ ] 可选「空间根目录」或「文件夹」

### 导入成功 toast
- [ ] 选好目标点确认 → toast `导入成功，文件正在解析中`
- [ ] 选择态自动退出
- [ ] 去知识空间页面看：刚导入的 .md 文件存在（文件名形如 `<会话标题>_yyyyMMddHHmm.md`）

### 同名追加序号（AC-19）
- [ ] 同一目录下连续导入两次同名会话 → 第二次 toast 文案 / 文件名含 `(1)`
- [ ] 第三次 → `(2)`

### 4 种格式导出（AC-02 / AC-10 / AC-11 / AC-12 / AC-13）
- [ ] 选好消息点「导出为 Word」→ 浏览器下载 `.docx`
- [ ] 文件名格式：`<会话标题>_yyyyMMddHHmm.docx`，中文标题正确解码
- [ ] 用 Word / WPS 打开 docx：
  - [ ] A4 纵向 + 白底
  - [ ] 中文字体清晰（不是小方块）
  - [ ] 用户名 / 模型名加粗成段
  - [ ] query 段与 answer 段之间一条横线分隔
  - [ ] Markdown 列表渲染为真实编号
  - [ ] 表格渲染为真实表格
  - [ ] 代码块带等宽字体 + 浅灰背景
  - [ ] 图片正确嵌入（不是大叉号）
- [ ] 「导出为 PDF」→ 下载 `.pdf`，PDF 阅读器打开样式与 docx 一致
- [ ] 「导出为 Markdown」→ 下载 `.md`，纯文本可读，图片用 `![](url)` 语法
- [ ] 「导出为 TXT」→ 下载 `.txt`，markdown 标记被剥除，表格降级为 Tab 对齐

### RAG 角标剥除（AC-14）
- [ ] 选一条含 RAG 引用角标 [1][2] 的回答导出
- [ ] 任何格式下：导出文件**不含** 私有 Unicode 角标 / 不含「[1]」「[2]」一类标记
- [ ] 末尾**不**追加「来源列表」

### 错误路径手测
- [ ] 选满 200 条 + 1 条 → toast 「单次最多选中 200 条消息」
- [ ] 对没有 upload_file 权限的知识空间导入 → toast 「暂无权限导入到该知识空间」（用别的账号造空间）
- [ ] 触发 LibreOffice 超时（导一份巨长会话）→ toast 「文件生成失败，请稍后重试」
