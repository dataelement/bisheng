# BiSheng 2.5 PRD vs Current Implementation Diff

更新时间：2026-04-23  
范围：仅记录 2.5 中的非租户功能差异。  
说明：按当前要求，**Tenant / 多租户 / Tenant 树 / Root-Child / 配额树 / 所有者交接**等内容不纳入项目文档。

## 1. 对照依据

主要 PRD：

- [`docs/PRD/2.5 权限管理体系改造 PRD/2.5 权限管理体系改造 PRD.md`](./PRD/2.5%20权限管理体系改造%20PRD/2.5%20权限管理体系改造%20PRD.md)

本轮重点核查模块：

- 组织与成员
- 用户组
- 角色与权限
- 部门知识空间
- 应用 / 工具授权
- 后台知识库（文档库 / QA 库）
- 前台工作台（应用中心 / 探索广场 / 知识空间 / 频道）
- 分享链路
- 组织同步
- 看板 / 频道 / 评测 / 微调 / 工作台补充扫描

## 2. 总览结论

### 已基本对齐

- 用户组管理的公开 / 私密可见性、创建者展示、成员编辑主流程基本已落地。
- 角色与权限里的 ReBAC 资源权限模板、前台资源授权弹窗、关系模型 CRUD 基本已落地。
- 部门知识空间的批量创建、审批开关、内容安全开关已有完整前端入口。
- 应用 / 工具的资源授权弹窗已接入，按钮级权限与资源级权限有明显区分。
- 前台知识空间侧边栏已把“部门知识空间”单独分区并置顶，基本符合 PRD。

### 部分对齐

- 组织与成员主界面已经是“左树右表”，但缺少若干 PRD 指定交互。
- 后台知识库中的“文档知识库”与“QA 知识库”实现不一致，前者已接 ReBAC 权限管理，后者未完全对齐。
- 前台应用中心 / 探索广场已具备列表与对话跳转，但“分享应用”按钮未体现独立的分享权限控制。
- 组织同步已有 Provider 配置、测试、手动执行、日志，但距离 PRD 里的同步范围 / 授权模式 / 执行计划还有缺口。

## 3.9 分享 / 公开访问链路

### 差异 13：应用分享更像“深链跳转”，不是稳定的 token 化公开分享

- PRD 语义：
  - 应用分享是独立能力
- 当前实现：
  - 应用分享链接由前端直接拼接：
    - [`src/frontend/client/src/pages/apps/appUtils.ts`](../src/frontend/client/src/pages/apps/appUtils.ts)
    - 格式：`/share/app_{appId}_{flowType}`
  - 分享落点由通用分享页处理：
    - [`src/frontend/client/src/pages/share.tsx`](../src/frontend/client/src/pages/share.tsx)
  - 对 `app_*` token 的处理逻辑是：
    - 解析出 `applicationId` / `flowType`
    - 直接跳到普通 `/app/...` 路由
    - **不会把 shareToken 传给应用详情 / 对话页**
- 影响：
  - 当前“应用分享”更像“便捷跳转到应用”的深链
  - 不等于真正的 token 化公开分享访问
- 优先级：P1

### 差异 14：`ShareAppRedirect.tsx` 已存在，但当前路由未使用

- 当前实现：
  - 存在文件：
    - [`src/frontend/client/src/routes/ShareAppRedirect.tsx`](../src/frontend/client/src/routes/ShareAppRedirect.tsx)
  - 但客户端路由表：
    - [`src/frontend/client/src/routes/index.tsx`](../src/frontend/client/src/routes/index.tsx)
  - 未看到它被挂到任何路由
  - 实际 `/share/:token` 走的是通用 [`src/frontend/client/src/pages/share.tsx`](../src/frontend/client/src/pages/share.tsx)
- 优先级：P2

- 看板分享：

## 4. 优先级建议

### P1

- 组织树搜索交互补齐
