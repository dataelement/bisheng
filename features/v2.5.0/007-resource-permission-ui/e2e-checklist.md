# E2E 验证清单: F007 Resource Permission UI

**测试环境**: http://192.168.106.114:3001 (Platform)
**前置条件**: 
- OpenFGA 服务运行 (localhost:8080)
- 至少两个用户: admin (super_admin) + 一个普通用户
- 至少一个部门和一个用户组存在

---

## Platform 前端

### AC-01: 权限管理对话框打开与结构
- [ ] 步骤 1: 以管理员登录 Platform
- [ ] 步骤 2: 导航到知识空间列表页
- [ ] 步骤 3: 点击某个知识库操作菜单（⋯）→ "权限管理"
- [ ] 预期: 弹出 Dialog，标题为"权限管理 - {知识库名称}"
- [ ] 验证: Dialog 包含 3 个 Tab：当前权限、添加权限、链接分享（disabled）
- [ ] 验证: 默认展示"当前权限"Tab

### AC-02: 当前权限列表展示
- [ ] 步骤: 打开权限管理 Dialog 的"当前权限"Tab
- [ ] 预期: 表格展示主体图标（用户/部门/用户组）、主体名称、权限级别、操作列
- [ ] 验证: owner 条目显示紫色徽章，无撤回按钮
- [ ] 验证: 非 owner 条目权限级别为 Select 下拉（可修改）
- [ ] 验证: 加载中显示 loading spinner

### AC-03: 通过"用户"方式授予权限
- [ ] 步骤 1: 打开 Dialog → "添加权限" Tab
- [ ] 步骤 2: 选择"用户"主体类型（默认选中）
- [ ] 步骤 3: 搜索框输入用户名，等待 300ms 后出结果
- [ ] 步骤 4: 勾选 1-2 个用户
- [ ] 步骤 5: 选择权限级别 viewer → 点击"提交"
- [ ] 预期: Toast "授权成功"，自动切换到"当前权限"Tab
- [ ] 验证: 刷新后列表中出现新授权的用户

### AC-04: 通过"部门"方式授予权限
- [ ] 步骤 1: "添加权限" Tab → 点击"部门"
- [ ] 步骤 2: 部门树展示，展开/折叠正常
- [ ] 步骤 3: 勾选一个部门，确认"包含子部门"已勾选
- [ ] 步骤 4: 选择 editor → 提交
- [ ] 预期: 授权成功，列表中出现部门条目（带"包含子部门"标注）

### AC-05: 通过"用户组"方式授予权限
- [ ] 步骤 1: "添加权限" Tab → 点击"用户组"
- [ ] 步骤 2: 用户组列表加载，支持搜索过滤
- [ ] 步骤 3: 勾选一个用户组 → manager → 提交
- [ ] 预期: 授权成功，列表中出现用户组条目

### AC-06: 修改权限级别
- [ ] 步骤: "当前权限"Tab → 非 owner 条目 → 点击权限级别 Select
- [ ] 步骤: 从 viewer 切换到 editor
- [ ] 预期: Toast "权限级别已修改"，列表刷新
- [ ] 验证: Network 面板显示 authorize 请求包含 revokes(旧) + grants(新)

### AC-07: 撤回权限
- [ ] 步骤: "当前权限"Tab → 非 owner 条目 → 点击撤回按钮（垃圾桶图标）
- [ ] 预期: 弹出确认框"确定要撤回该权限吗？"
- [ ] 步骤: 点击确认
- [ ] 预期: Toast "已撤回权限"，条目从列表中消失

### AC-08: 资源列表页权限徽章
- [ ] 步骤: 知识空间列表页
- [ ] 预期: 每个知识库名称旁显示权限徽章（紫=owner, 蓝=manager, 绿=editor, 灰=viewer）
- [ ] 步骤: 应用列表页
- [ ] 预期: 卡片标题旁显示权限徽章
- [ ] 步骤: 工具列表页
- [ ] 预期: 工具名称旁显示权限徽章
- [ ] 步骤: 仪表盘侧栏
- [ ] 预期: 仪表盘名称旁显示权限徽章

### AC-09: 操作按钮权限控制（过渡期）
- [ ] 步骤: 以 admin 登录 → 验证所有页面权限管理入口可见
- [ ] 步骤: 以普通 viewer 用户登录
- [ ] 预期: 权限管理入口不可见（Shield 图标/菜单项不渲染）
- [ ] 注意: 编辑/删除按钮仍由 data.write 控制（过渡期不变）

### AC-11: 主体搜索交互
- [ ] 用户搜索: 输入后 300ms 防抖 → loading → 结果列表可滚动
- [ ] 部门树: 展开/折叠流畅，搜索过滤节点高亮
- [ ] 用户组: 搜索过滤实时响应
- [ ] 空结果: 显示"无匹配结果"友好提示

### AC-12: 权限管理入口可见性
- [ ] 以 owner/manager 登录 → 4 个页面的权限管理入口可见
- [ ] 以 editor/viewer 登录 → 权限管理入口完全不渲染（不是 disabled）
