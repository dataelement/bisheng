
1. ### 知识空间 AI 问答- 知识检索权限过滤
  

1. #### **背景**
  

当前知识空间支持空间、文件夹、文件三个层级的权限控制，但 AI 问答在检索知识空间内容时，不能只判断用户是否能进入空间，还必须保证最终进入模型上下文的文件内容，也来自当前用户有权限查看的文件。

本需求目标是：**所有使用知识空间内容进行 AI 问答的场景，都必须按当前登录用户的知识空间、文件夹、文件权限过滤检索范围，避免用户通过 AI 问答读到无权访问的文件内容、文件名、摘要或引用来源。**

2. #### **需求范围**
  

|     |     |
| --- | --- |
| 场景  | 说明  |
| 知识空间整空间问答 | 用户在知识空间页面，使用AI助手对整个空间提问 |
| 知识空间文件夹问答 | 用户在某个文件夹页面，使用AI助手对文件夹范围内提问 |
| 文件预览页问答 | 用户选中某个文件，使用AI助手针对某个文件提问 |
| 首页/工作台问答 | 用户在首页选择知识空间后提问 |

3. #### **权限原则**
  

本需求统一采用“当前登录用户”作为权限判断主体，复用ReBAC权限体系。

|     |     |
| --- | --- |
| 对象  | 必要权限 |
| 知识空间 | `view_space` |
| 文件夹 | `view_folder` |
| 文件  | `view_file` |
| AI 检索 chunk | chunk 所属文件必须满足 `view_file` |
| 回答引用来源 | 来源文件必须满足 `view_file` |

核心原则：

1. AI 检索最终以“可见文件集合”为边界。
  
2. 用户无权限查看的内容不得进入模型上下文，也不得出现在引用、来源、错误提示、日志返回给前端的结构化字段中。
  
3. 权限变化后，新一轮问答必须立即按最新权限生效。
  

4. #### **场景规则**
  

|     |     |     |
| --- | --- | --- |
| 场景  | 检索范围规则 | 无权限处理 |
| 整空间问答 | 空间内当前用户有 `view_file` 的成功解析文件 | 不可见无权限的知识空间。<br>没有可用文件时，AI助手检索不到内容，返回内容由提示词决定<br><br><br> |
| 文件夹问答 | 目标文件夹下当前用户有 `view_file` 的成功解析文件<br><br><br> | 不可见无权限的文件夹。<br>没有可用文件时，AI助手检索不到内容，返回内容由提示词决定 |
| 文件预览页问答 | 当前文件必须有 `view_file` | 不可见该文件 |
| 首页/工作台问答 | 每个被选中的知识空间分别过滤可见文件<br><br><br> | 对知识空间没有`view_space`权限，不可见该知识空间（下拉框中不可见）。<br><br>对知识空间有权限，知识空间内的无权限文件需要被跳过。<br><br>若知识空间内均为无权限文件，返回内容由提示词决定。 |
| 继续对话 | 新问题检索时重新按当前权限过滤<br><br><br> | 文件、文件夹权限被收回后，需要筛选掉相关文件的内容。<br>对知识空间失去权限后，后端取消掉对该知识空间的检索 |

5. #### 设计要求
  

1. ##### 复用现有权限体系
  

权限判断继续复用现有 ReBAC / 细粒度权限体系：

- knowledge_space: view_space
  
- folder: view_folder
  
- knowledge_file: view_file
  

2. ##### 性能要求
  

避免逐 chunk 查询权限。

避免对空间内文件逐个串行查询 OpenFGA。

筛选文件不会对正常业务的响应速度造成明显影响。

当空间文件量较大时，应保证权限过滤不会导致问答接口超时。

每次新问题、继续对话都要重新计算可检索文件集合，保证实时性。

6. #### **异常与边界**
  

|     |     |
| --- | --- |
| 情况  | 期望行为 |
| 问答过程中权限被收回 | 下一轮检索按新权限生效 |
| 历史回答引用了后来无权的文件 | 历史消息可保留，但再次查看引用详情时应按当前权限校验 |
| 选中的知识空间存在文件无权限<br><br><br> | 有权限的参与检索，无权限的跳过不参与检索 |
| 文件解析失败/处理中 | 不参与检索 |
