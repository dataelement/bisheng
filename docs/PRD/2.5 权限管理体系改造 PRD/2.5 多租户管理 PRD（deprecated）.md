<tr>
<td></td>
<td>租户管理</td>
<td><img src="images/2.5%20%E5%A4%9A%E7%A7%9F%E6%88%B7%20PRD-image-3.png" alt=""><img src="images/2.5%20%E5%A4%9A%E7%A7%9F%E6%88%B7%20PRD-image-4.png" alt=""><ul>
<li>
<p>目标用户：仅平台超级管理员可见此菜单。</p>
</li>
<li>
<p>初始化：初始化系统时，首个登录的用户为平台超级管理员。进入系统后，先弹出新建租户的页面，平台超级管理员需要建立一个租户，才能进行后续的操作。</p>
</li>
<li>
<p>租户ID：系统自动生成租户的唯一标识，以<code>&#x3C;str>t_xxx_xxx&#x3C;/str></code> 格式展示。</p>
</li>
<li>
<p>租户/企业名称：必填，最大长度50字符，不支持特殊表情符号。</p>
</li>
<li>
<p>租户超级管理员：租户超级管理员即本PRD中广泛提及的超级管理员。账号即人员ID，是登录唯一凭证，不可修改，需校验全局唯一性。</p>
</li>
<li>
<p>设置新密码：可以更改超级管理员的密码。</p>
</li>
<li>
<p>存储配额：</p>
<ul>
<li>数值呈现：实时统计该租户下所有知识空间文件总容量作为“已用量”。必填，默认值为100.</li>
<li>进度条：根据 (已用量/总配额)*100% 渲染。</li>
</ul>
</li>
<li>
<p>启用：当平台管理员在列表页将某个租户的开关拨至“关闭”时：</p>
<ul>
<li>
<p>二次确认：弹出二次确认框：“停用后，该企业下所有成员将被立即强制下线，且无法再次登录系统。是否确认停用？”</p>
</li>
<li>
<p>后端动作：</p>
<ol>
<li>立即吊销该 tenant_id 下所有活跃账号的 JWT/Session Token，触发全员强制登出。</li>
<li>接口网关层拦截所有携带该 tenant_id 的请求，返回 403 Forbidden ，提示企业已停用。</li>
<li>定时同步任务针对该租户立即挂起暂停。</li>
</ol>
</li>
</ul>
</li>
<li>
<p>存储配额超限：当某租户的物理文件上传导致其“当前存储配额”达到 100% 时，该租户内的所有成员（包括其超管）在调用“上传文件”、“新建知识库”等占用存储空间的接口时，后端一律阻断拦截。前端提示：当前企业存储配额已耗尽（XX/XX GB），请联系平台方进行配额扩容。</p>
</li>
</ul></td>
</tr>
