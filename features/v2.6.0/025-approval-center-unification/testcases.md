# 审批中心 测试用例（F025）

> **版本**：v2.6.0
> **关联规格**：[spec.md](./spec.md)
> **测试范围**：审批网关、审批流程运行时、管理配置页、用户端（我的审批/我的申请）、三个主场景、权限隔离、站内信

---

## 测试账号约定

| 角色 | 说明 |
|------|------|
| `超管` | 系统管理员，`is_admin=True` |
| `租管` | 租户管理员 |
| `部门管` | 部门 A 的部门管理员 |
| `申请人` | 普通用户，无特殊权限 |
| `审批人` | 已在流程节点中配置为审批人的用户 |
| `无关用户` | 与当前审批实例无关联的普通用户 |

**前置假设**：系统中已配置 `menu_access_request` / `channel_subscribe_request` / `knowledge_space_subscribe_request` 三个预置场景（研发侧已注册），工作台开启 `menu_approval_mode=true`。

## A. 审批网关行为

### TC-A01 场景未配置时调用业务接口

| 项 | 内容 |
|----|------|
| **前置** | 审批管理中未添加 `menu_access_request` 场景 |
| **步骤** | 申请人在无权限菜单占位页点击"申请权限" |
| **预期** | 页面显示"此功能未开放申请"；后端不创建任何 `approval_instance` 或 `approval_exception` |

### TC-A02 场景已配置但未启用时调用业务接口

| 项 | 内容 |
|----|------|
| **前置** | 审批管理中已添加 `menu_access_request` 场景，但开关为**停用** |
| **步骤** | 申请人在无权限菜单占位页点击"申请权限" |
| **预期** | 页面显示"此功能未开放申请"；后端不创建任何 `approval_instance`，不写 `approval_exception` |

### TC-A03 场景已启用且命中"无需审批"分支

| 项 | 内容 |
|----|------|
| **前置** | `menu_access_request` 场景已启用；管理员配置分支规则：申请人身份=超管 → 无需审批 |
| **步骤** | 以**超管**账号访问无权限占位页并提交申请 |
| **预期** | 后端创建 `approval_instance`（status=executed）；不生成 `approval_task`；业务动作（写个人菜单授权）直接执行；写 `approval.route.pass` 审计 |

### TC-A04 场景已启用且命中审批流程分支

| 项 | 内容 |
|----|------|
| **前置** | `menu_access_request` 场景已启用；配置分支：普通用户 → 流程 A（节点1：部门管理员审批） |
| **步骤** | 以**申请人**提交菜单权限申请 |
| **预期** | 创建 `approval_instance`（status=pending）；创建第一节点 `approval_task`（分配给部门管）；返回 `PENDING`；前端提示"申请已提交"；审批人收到站内信 |

### TC-A05 重复提交同一业务对象

| 项 | 内容 |
|----|------|
| **前置** | 申请人已对某菜单提交审批且处于 pending 状态 |
| **步骤** | 申请人再次对同一菜单点击"申请权限"并提交 |
| **预期** | 后端不创建新实例；返回已有实例 ID；前端提示"已有审批中申请"并提供跳转链接 |

## B. 审批流程运行时

### TC-B01 单节点单审批人同意

| 项 | 内容 |
|----|------|
| **前置** | 流程仅1节点，审批人为部门管；审批实例 pending |
| **步骤** | 审批人在"我的审批"点击同意，可填审批意见 |
| **预期** | 任务状态→ approved；实例状态→ approved → executing → executed；业务动作执行（写授权）；申请人收到成功通知 |

### TC-B02 单节点单审批人拒绝

| 项 | 内容 |
|----|------|
| **前置** | 同 TC-B01 |
| **步骤** | 审批人点击拒绝，**不填原因直接提交** |
| **预期** | 接口返回错误（拒绝原因必填）；任务状态不变 |

### TC-B03 拒绝（填写原因）

| 项 | 内容 |
|----|------|
| **前置** | 同 TC-B01 |
| **步骤** | 审批人点击拒绝，填写原因后提交 |
| **预期** | 任务状态→ rejected；实例状态→ rejected；业务动作不执行；申请人收到拒绝通知（含原因摘要） |

### TC-B04 节点或签——一人同意即通过

| 项 | 内容 |
|----|------|
| **前置** | 节点配置两个审批人（张三、李四），节点模式=或签 |
| **步骤** | 张三同意 |
| **预期** | 张三任务→ approved；李四任务→ skipped；节点通过；若为最后节点则实例→ approved；后续流程继续 |

### TC-B05 节点会签——必须全部同意才通过

| 项 | 内容 |
|----|------|
| **前置** | 节点配置两个审批人（张三、李四），节点模式=会签 |
| **步骤** | ①张三同意 → 查看实例状态；②李四同意 |
| **预期** | 步骤①：张三任务→ approved，李四任务仍 pending，实例保持 pending；步骤②：李四任务→ approved，节点通过，实例→ approved |

### TC-B06 多节点顺序流转

| 项 | 内容 |
|----|------|
| **前置** | 流程2个节点：节点1=部门管审批（或签），节点2=超管审批（单人） |
| **步骤** | ①节点1审批人同意 → ②节点2审批人同意 |
| **预期** | 步骤①完成后才生成节点2任务；节点2审批人才会收到站内信；步骤②完成后实例→ executed |

### TC-B07 申请人撤回审批中申请

| 项 | 内容 |
|----|------|
| **前置** | 审批实例处于 pending 状态，已有节点1任务生成 |
| **步骤** | 申请人在"我的申请"点击"撤回申请" |
| **预期** | 所有 pending 任务→ cancelled；实例→ withdrawn；审批人收到"申请已撤回"通知；申请人的我的申请列表显示"已撤回" |

### TC-B08 申请被拒绝后重新提交

| 项 | 内容 |
|----|------|
| **前置** | 审批实例处于 rejected 状态 |
| **步骤** | 申请人在"我的申请"点击"重新提交" |
| **预期** | 创建**新的** `approval_instance`（status=pending）；旧实例保持 rejected 历史记录不变；新申请重新走审批流程 |

### TC-B09 拒绝后重新提交——非申请人操作

| 项 | 内容 |
|----|------|
| **前置** | 审批实例处于 rejected 状态 |
| **步骤** | **无关用户**尝试调用重新提交接口（知道 instance_id） |
| **预期** | 接口返回权限拒绝错误 |

## C. 异常流程处理

### TC-C01 条件分支未命中（route_missing）

| 项 | 内容 |
|----|------|
| **前置** | 场景已启用，但分支规则未覆盖当前申请人身份（无兜底分支） |
| **步骤** | 申请人提交申请 |
| **预期** | 实例→ exception（route_missing）；不生成任务；管理员收到站内信；异常列表显示该记录；申请人"我的申请"显示处理中 |

### TC-C02 审批人解析为空（approver_empty）

| 项 | 内容 |
|----|------|
| **前置** | 节点配置"申请人部门管理员"为审批人来源，但申请人无所属部门 |
| **步骤** | 申请人提交申请 |
| **预期** | 实例→ exception（approver_empty）；管理员收到站内信；异常列表显示 |

### TC-C03 业务执行失败（execute_failed）

| 项 | 内容 |
|----|------|
| **前置** | 审批节点已全部通过；业务 handler 执行时抛出异常（如菜单 key 已被删除） |
| **步骤** | 等待 outbox worker 执行 |
| **预期** | 实例→ execute_failed；创建 `approval_exception`（execute_failed）；管理员收到通知；申请人"我的申请"显示失败原因；审批结论不回退 |

### TC-C04 管理员处理 route_missing：指定流程继续

| 项 | 内容 |
|----|------|
| **前置** | 存在 route_missing 异常实例（TC-C01） |
| **步骤** | 管理员在异常流程列表选择该记录 → 点击"指定流程继续" → 选择一个已有流程 |
| **预期** | 生成该流程第一节点的 `approval_task`；实例→ pending；审批人收到通知；异常记录→ resolved |

### TC-C05 管理员处理 approver_empty：指定审批人继续

| 项 | 内容 |
|----|------|
| **前置** | 存在 approver_empty 异常实例（TC-C02） |
| **步骤** | 管理员在异常列表 → 指定用户 A 为审批人 → 点击继续 |
| **预期** | 为用户 A 生成当前节点 `approval_task`；实例→ pending；用户 A 收到通知 |

### TC-C06 管理员重试 execute_failed

| 项 | 内容 |
|----|------|
| **前置** | 存在 execute_failed 异常实例（TC-C03） |
| **步骤** | 管理员在异常列表 → 点击"重试执行" |
| **预期** | handler 重新执行；成功则实例→ executed，重试次数累计；失败则保留 execute_failed 并追加错误日志；审批结论不回退 |

### TC-C07 管理员取消任意异常审批

| 项 | 内容 |
|----|------|
| **前置** | 存在任意异常实例 |
| **步骤** | 管理员点击"取消审批"，**不填取消原因直接提交** |
| **预期** | 提交失败（取消原因必填） |

### TC-C08 管理员取消（填写原因）

| 项 | 内容 |
|----|------|
| **前置** | 同 TC-C07 |
| **步骤** | 管理员填写取消原因后提交 |
| **预期** | 实例→ cancelled；所有 pending 任务关闭；业务不再执行；写审计日志（含取消原因）；申请人收到通知 |

## D. 审批管理页（管理员配置）

### TC-D01 新增审批场景

| 项 | 内容 |
|----|------|
| **前置** | 以租管登录 → 审批管理页 |
| **步骤** | 点击"新增" → 场景名称下拉选择"菜单权限申请" → 保存 |
| **预期** | 列表出现新场景（默认停用）；场景编码、业务 handler 为只读展示；无法手动填写 |

### TC-D02 重复新增相同场景

| 项 | 内容 |
|----|------|
| **前置** | `menu_access_request` 场景已存在 |
| **步骤** | 再次点击"新增"，尝试选择相同场景 |
| **预期** | 下拉中不展示已创建的场景；若强行调用接口，后端返回 `ApprovalScenarioDuplicateError`（18105） |

### TC-D03 启用 / 停用场景

| 项 | 内容 |
|----|------|
| **步骤** | ①切换场景为"启用" → ②再次切换为"停用" |
| **预期** | 步骤①：业务入口可走审批流程；步骤②：业务入口调用时返回"此功能未开放申请" |

### TC-D04 删除场景

| 项 | 内容 |
|----|------|
| **步骤** | 删除某个已配置场景 |
| **预期** | 场景从列表移除；后续业务请求视为"未配置"并报错；已发起的历史实例数据不受影响 |

### TC-D05 场景保存后不可切换类型

| 项 | 内容 |
|----|------|
| **前置** | `menu_access_request` 场景已保存 |
| **步骤** | 点击编辑，尝试在下拉中切换为另一个预置场景 |
| **预期** | 接口拒绝修改 `scenario_code`；管理员须删除后重新创建 |

### TC-D06 新增条件分支

| 项 | 内容 |
|----|------|
| **步骤** | 在场景配置中新增分支：申请人身份=超管 → 无需审批 |
| **预期** | 分支保存成功；之后超管提交申请时命中该分支，直接通过 |

### TC-D07 条件分支排序（上下移动）

| 项 | 内容 |
|----|------|
| **前置** | 场景下有3条分支（A/B/C，按序匹配） |
| **步骤** | 将分支 B 上移至第一位 |
| **预期** | 保存后分支顺序为 B/A/C；新提交申请按新顺序匹配（命中 B 停止） |

### TC-D08 删除条件分支 / 启停分支

| 项 | 内容 |
|----|------|
| **步骤** | ①停用分支 B → 提交新申请；②删除分支 B → 提交新申请 |
| **预期** | 步骤①：匹配时跳过 B，继续匹配后续；步骤②：B 从列表消失，新申请不再经过 B |

### TC-D09 新建审批流程及节点

| 项 | 内容 |
|----|------|
| **步骤** | 在分支"进入审批流程"处填写流程名称 → 在审批流程区域选择该流程 → 新增两个节点（节点1：或签，节点2：会签） |
| **预期** | 流程及节点保存成功；可在"流程预览"弹窗看到节点顺序、审批人来源和签类型 |

### TC-D10 修改流程节点不影响已发起实例

| 项 | 内容 |
|----|------|
| **前置** | 已有申请实例基于流程版本 v1 处于 pending |
| **步骤** | 管理员修改该流程（删除一个节点），生成版本 v2 |
| **预期** | 已有实例继续按 v1 流转；新申请使用 v2；审批中实例的进度展示不受影响 |

### TC-D11 普通用户无法访问审批管理页

| 项 | 内容 |
|----|------|
| **步骤** | 以普通用户身份访问 `/approval` 管理路由 |
| **预期** | 返回 403；不展示场景配置内容 |

## G. 业务场景

### TC-G01 菜单权限申请——完整流程

| 项 | 内容 |
|----|------|
| **前置** | `menu_approval_mode=true`；`menu_access_request` 场景已启用；申请人无"审批管理"菜单权限 |
| **步骤** | ①申请人访问审批管理菜单 → 看到无权限占位页 → 点击"申请权限" → 填写理由 → 提交；②审批人在"我的审批"同意 |
| **预期** | 步骤①：创建审批实例，页面提示"申请已提交"；步骤②：写入 `user_menu_access` 个人授权，申请人刷新页面后可访问该菜单 |

### TC-G02 菜单授权只写个人，不影响角色菜单

| 项 | 内容 |
|----|------|
| **步骤** | 审批通过后，查看该申请人所属角色的菜单权限配置 |
| **预期** | 角色菜单配置不变；只有申请人的 `user_menu_access` 中写入了新记录；其他同角色用户仍无该菜单权限 |

### TC-G03 父级依赖菜单自动补齐

| 项 | 内容 |
|----|------|
| **步骤** | 申请访问需要父级入口的子菜单，审批通过 |
| **预期** | `user_menu_access` 中同时写入子菜单和父级入口菜单授权，用户刷新后能正常进入子菜单 |

### TC-G04 审批人撤回菜单授权

| 项 | 内容 |
|----|------|
| **前置** | 某菜单权限申请已 executed（授权已生效） |
| **步骤** | 历史审批人在"我的审批"已处理记录中，点击"撤回授权" → 填写理由 |
