# 阴影与圆角规范 Elevation

阴影与圆角共同塑造界面的层次：圆角决定一块表面看起来有多柔和，阴影决定它「浮」得有多高。两者都只从固定的档位里取值——圆角八档、投影两档，按场景选档，不自造数值。

## 1. 圆角 Radius

界面里的圆角分两类：跟着控件尺寸走的**控件档**，和跟着容器大小走的**容器档**。先判断你在做哪一类，再按下面的阶梯取值。

### 1.1 阶梯一览

<RadiusLadder />

类名即 token 名（`rounded-sm` 就是 4px 档），完整的 CSS 变量与 Tailwind 类对照见「设计变量 Design Token」页。

### 1.2 控件圆角：跟尺寸档走

- **控件（按钮、输入框、选择器、菜单项）的圆角随自身尺寸取 sm / md / lg（4 / 6 / 8px）**：small 控件用 4px、medium 用 6px、large 用 8px。同一行里的控件尺寸档一致，圆角自然对齐。按钮三档即照此取（见「组件 → Button」）。
- **拿不准就用 md（6px）**——它是 medium 控件的默认档，也是界面里最常见的控件圆角。
- **胶囊与圆形用 full**：头像、pill 标签、圆形 icon 按钮。圆形按钮仅限单 icon 按钮，限制见「组件 → Button」。

### 1.3 容器圆角：跟容器大小走

- **卡片、下拉面板用 xl（12px）；弹窗、大卡片容器用 2xl（16px）；抽屉、超大容器用 3xl（24px）；特大区块用 4xl（32px）**。容器越大，圆角越大，浮起的层看起来才不生硬。
- **弹窗统一 16px**。

### 1.4 贴边的角不留圆角

圆角属于「浮在内容之上的独立表面」。同一个容器换到移动端时，如果它铺满屏幕或贴住了屏幕边缘，贴边的角就改用直角——不是换一档更小的圆角，而是贴边侧归零：

- **移动端全屏展示**：四角全部直角。例如一个弹窗在 PC 端悬浮时用 12px，到移动端整页铺满，四角就改直角。
- **移动端从底部滑入贴底**：只保留顶部两角的容器档圆角，底部两角直角。
- 悬浮形态不变的容器（移动端仍居中、四周留边）圆角照 §1.3 取，不因屏幕变窄而缩档。

多端断点与判定口径见「多端适配 Responsive」，本条是圆角侧的细则。

### 1.5 嵌套圆角

**内层圆角 = 外层圆角 − 内边距**（结果归并到最近档位，最小 4px），这样内外两段圆弧同心，角部间隙均匀。例如弹窗（16px）内边距 8px 时，贴边卡片用 8px；内层照抄外层的圆角，四角就会挤出月牙形空隙。

### 1.6 正误对照

| ✅ 推荐 | ❌ 不推荐 | 原因 |
|---|---|---|
| `rounded-2xl`（弹窗 16px） | `rounded-[20px]` | 20px 不在阶梯上；同类弹窗一个 16 一个 20，并排就对不齐 |
| `rounded-sm`（4px） | `rounded-[5px]`、`rounded-[0.185rem]` | 离散值与 4px 肉眼难辨，却让「同一控件同一圆角」失效 |
| `rounded-sm` | `rounded`（裸类） | 两者同为 4px，但裸类不带 token 名，统一写 `rounded-sm` 才看得出档位 |

## 2. 阴影 Shadow

BISHENG 的投影走「淡」路线：带一点蓝黑色、透明度很低，负责把浮层轻轻托起来，而不是砸出一圈黑晕。投影只有两档，按元素浮起的高度取。

### 2.1 投影两档

| 档位 | 投影值 | Tailwind 类（即 token 名） | 用在哪 |
|---|---|---|---|
| 浮层投影 Popup | `0 2px 16px -2px rgba(0,23,66,0.10)` | `shadow-popup` | 下拉菜单、选择器面板、气泡卡片、悬浮提示——点开即现、点外即走的临时浮层 |
| 模态投影 Modal | `0 0 16px 0 rgba(3,7,117,0.05)` | `shadow-modal` | 弹窗、抽屉——要求用户处理完才关闭的打断式浮层 |

<ShadowTiers />

档位与类名是同一个 token 的两面：token 存「名字 + 值」，Tailwind 类名是代码里取用它的写法，两者同名同物。业务代码取用类名即可，不要自己写投影值。

### 2.2 使用规则

- **投影颜色用蓝黑，不用纯黑**：两档的投影色都带一点蓝、透明度不超过 10%。纯黑投影在浅色界面上发灰发脏，和淡投影混排还会打乱层次。按两档取值，这条自动满足。
- **页面内静止的区块不加投影**：普通卡片、表格、表单区用边框和填充分层。只有真正浮在内容之上的元素（下拉、气泡、弹窗、抽屉）和拖拽中的元素才带投影——什么都浮起来，等于什么都没浮起来。
- **只用这两档，不自造投影**：两档之外的任何投影——包括 Tailwind 自带的预设投影档和手写的投影值——都不使用。第三方组件自带的投影可以整个去掉，但不能改成别的值。
- **投影可以和 1px 浅色边框搭配**：弹窗常用「淡投影 + 边框」的组合，暗色模式下投影不明显时边框还能帮浮层勾出轮廓。是否带边框由各组件自定，不作硬性要求。

### 2.3 正误对照

<ShadowContrast />

| ✅ 推荐 | ❌ 不推荐 | 原因 |
|---|---|---|
| 下拉面板：`shadow-popup`（浮层投影） | `shadow-lg`（Tailwind 纯黑预设） | 一次违反两条规则：投影不在两档内，颜色是纯黑——又重又灰，和蓝黑淡投影混排会打乱层次 |
| 静态卡片：只用边框分层 | 页面里每张卡都加投影 | 投影表达的是「浮起的高度」，静态内容浮起来没有意义，还让真正的浮层失去对比 |

## 3. 使用约定

- **圆角与投影一律取表内档位**：出现表外的手写值即视为未走规范；确属例外的，在对应组件文档里写明原因并注明「见阴影与圆角规范」。
- **新档位先进规范、再进代码**：要用新的圆角或投影值，先由设计师在本文与 design token 里定档，之后才能出现在代码里，不允许先用后补——这就是本站所说「规范评审」的全部含义：设计师定夺 + 文档与 token 先行。
