# E2E 验证清单: F033 部门知识空间授权范围

**测试环境**: Client 端（End-user chat UI）`/workspace` — 知识空间页
**前置条件**:
- 已有一个**部门知识空间**（绑定某部门，例如「研发部的知识空间」），且该部门有子部门与成员。
- 已有一个**普通知识空间**（非部门绑定）。
- 以该空间的**创建者/管理员**登录；另准备一个 **super_admin** 账号做不豁免验证。

> 后端行为（部门/人员收敛、用户组拒绝、超管不豁免、普通空间不变）已由
> `test/e2e/test_e2e_department_space_scope.py`（10 passed）+ `test_e2e_resource_permission_ui.py`（13 passed）覆盖。
> 本清单只验证**前端 UI**（AC-01）与端到端观感。

## Client 前端

### AC-01: 部门知识空间 — 新增授权只有「用户 / 部门」两个 tab
- [ ] 步骤 1: 进入某**部门知识空间** → 打开「成员管理 / 权限管理」弹窗
- [ ] 步骤 2: 点击「新增授权 / 授权」
- [ ] 预期: subject 维度只有「**用户**」「**部门**」两个 tab，**没有「用户组」**
- [ ] 步骤 3: 看权限列表区的维度切换
- [ ] 预期: 列表区同样**没有「用户组」** tab（与授权弹窗同源）
- [ ] 步骤 4: 「部门」tab 下展开部门选择
- [ ] 预期: 只能看到「该空间绑定的部门 + 其子部门」，看不到其它部门
- [ ] 步骤 5: 「用户」tab 下搜索/浏览
- [ ] 预期: 只能看到上述部门子树内的成员

### AC-01 (super_admin 不豁免): 超管在部门空间同样只有两 tab
- [ ] 步骤: 以 **super_admin** 登录，重复上面的步骤 1-2
- [ ] 预期: 仍然只有「用户 / 部门」两个 tab，无「用户组」；部门/人员范围同样被收敛

### AC-04 (前端观感): 越界授权被后端拒绝并提示
- [ ] 步骤: （若可构造）尝试授权一个子树外的对象 / 或直接观察正常授权
- [ ] 预期: 子树内用户授权成功；若发生越界，toast 提示「只能授权给本部门及子部门的成员 / 不支持按用户组授权」（后端 PermissionDenied 文案）

### AC-05 (回归): 普通知识空间三 tab 不变
- [ ] 步骤 1: 进入某**普通知识空间** → 打开权限弹窗 → 新增授权
- [ ] 预期: subject 维度有「**用户 / 部门 / 用户组**」三个 tab（用户组仍在）
- [ ] 步骤 2: 「部门」tab
- [ ] 预期: 展示**全租户**部门树（不被收敛）
- [ ] 步骤 3: 用「用户组」tab 授权一个用户组
- [ ] 预期: 授权成功，列表出现该用户组

## 回归检查
- [ ] 部门空间 / 普通空间的权限弹窗都能正常打开，无 console 错误
- [ ] 成员列表、角色升降、移除等既有功能不受影响
- [ ] 管理员 / super_admin / 普通成员看到的内容符合各自权限
