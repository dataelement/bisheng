# 技术评审：知识空间 文件 / 文件夹移动（同空间 + 跨空间）

> 把文件 / 文件夹移动到本空间内别的目录、**或别的知识空间**——支持「移动到」弹窗和多选拖拽两种方式。
> （v2：原评审只做同空间；产品已拍板跨空间也做，并给了简化条件，本文已更新。）

---

## 一、范围:这次做什么、不做什么

| | 内容 |
|---|---|
| ✅ 本次做 | 同空间移动（移动到弹窗 + 多选拖拽）；**跨空间移动（文件和文件夹都支持，弹窗与拖拽两种方式均可）**；新增「移动文件 / 移动文件夹」两个权限；循环 / 层级 / 重名（仅同空间）/ 权限校验；批量部分失败处理；同空间移动成功后可「撤回」 |
| ❌ 本次不做 | 跨空间的重名校验、标签迁移（移动后标签清空）、撤回（以**二次确认**替代）；移动过程不重新解析文档 |

---

## 二、整体思路:两条路径，一个入口

**同空间 = 只改"归属"**（秒级、可撤回）：

```
选中 + 选目标 → 逐项校验（权限/循环/层级/重名）
  → 改"层级归属" + 换"权限父节点"        ← 纯元数据
  → 列表刷新 + "移动成功"toast（带撤回）
```

**跨空间 = 归属即时改 + 检索数据后台搬**（二次确认、无撤回）：

```
选中 + 选别的空间（弹窗选 / 拖到左侧空间上）→ 二次确认 → 校验（权限/层级,不查重名）
  → 一个事务里：文件连同它的全部历史版本一起改归属 + 换权限父节点
     + 已解析完成的文件标记「处理中」+ 清空空间标签
  → 用户立刻在新空间看到文件 + 成功 toast
  → 后台任务逐个文件搬检索数据：
     从旧空间把文本块读出来 → 按新空间的向量模型重新计算 → 写入新空间 → 删旧空间数据
  → 搬完恢复正常（可被检索）；搬失败标记失败、可重试
```

**为什么跨空间可行了？** 三个关键事实（本次调研确认）：
1. **文件本体不用搬**——MinIO 对象按文件 id 存，不按空间存，改归属即可。
2. **系统已有同构管线**——「知识库复制」功能就是这么搬向量的，且写入时自动用目标空间的向量模型重算，天然兼容两空间模型不同。
3. **不用重新解析**——文本块在 ES 里是全的，直接作为搬运源，省掉最贵的 ETL。

## 三、产品已确认的决策

| 决策 | 结论 |
|---|---|
| 搬运窗口怎么呈现 | 文件立刻出现在新空间列表,状态显示「处理中」;搬完前内容暂搜不到（接受） |
| 文件夹能否跨空间移 | **能**——子树内全部文件（含历史版本）逐个搬检索数据 |
| 解析中/排队中文件跨空间移 | 移过去后**在新空间继续完成解析**（解析任务执行时按文件当前归属落库） |
| 目标侧权限 | **仅弹窗展示过滤**（只列有上传权限的空间/文件夹），服务端不重复校验目标权限 |
| 跨空间成功反馈 | toast 提示移动成功（无撤回入口） |
| 空间标签 | 跨空间移动后**清空**文件原有标签 |
| 拖拽范围 | 同空间（拖到可视文件夹行）和跨空间（拖到左侧空间列表项=移到该空间根，松手后二次确认）**都支持** |

## 四、要改哪些地方

| 层 | 改动 | 量级 |
|---|---|---|
| 后端 - 权限配置 | 加「移动文件 / 移动文件夹」两行映射（不动权限模型、不迁数据） | 极小 |
| 后端 - 接口 | 1 个移动接口 `POST /{空间}/files/move`（带目标空间参数,同/跨空间自动分流） | 小 |
| 后端 - 业务 | 移动编排：校验 + 改层级路径 + 换权限父节点 + 文件夹级联子树 + **跨空间时版本链整链改归属 + 清标签** | 中 |
| 后端 - 异步任务 | 新增「搬运检索数据」任务（ES 读块 → 新空间重嵌入写入 → 删旧 → 图片目录拷贝）+ 状态流转(处理中→成功/失败可重试) | 中 |
| 后端 - 错误码 | 加 1 个「无效移动目标」错误码(18033),其余复用 | 极小 |
| 前端 - 弹窗 | 「移动到」弹窗：**左侧空间列表（只列有上传权限的）**、右侧文件夹树+文件（无权限置灰,文件置灰） | 中 |
| 前端 - 拖拽 | 列表行多选拖拽（文件夹行 + 左侧空间项两类投放目标）;拖拽中不展开文件夹/不进空间 | 中 |
| 前端 - 交互 | 批量冲突弹窗、同空间成功 toast 带撤回、**跨空间二次确认 + 成功 toast**、「处理中」状态显示 | 中 |

## 五、风险 / 注意点

| 项 | 说明 |
|---|---|
| 版本链整链迁移 | 移动必须把主版本+全部历史版本一起改归属（系统强制"同一版本链必须同空间"）,漏了历史版本会直接报错。历史版本各有自己的检索数据,也要逐个搬。 |
| 子树级联 | 移动文件夹必须连带重写所有子孙层级路径;跨空间还要逐个改归属+派发搬运。 |
| 深度校验 | 「移动后不超 10 层」按**被移子树的最深层**算。 |
| 搬运窗口 | 跨空间搬完之前,文件在新空间「看得见、搜不到」——产品已接受,状态显示「处理中」。 |
| 图片目录 | 文档里抽出的图片按**空间目录**存,移动后引用仍指旧空间;不搬将来删旧空间会把图弄丢——实现时连图片一起搬。 |
| 解析中竞态 | 解析任务执行中途文件被移走,有极小窗口产物落旧空间——实现时在解析完成处回查归属兜底。 |
| 双 DB | 子树批量前缀重写避免 MySQL-only SQL（REPLACE/CONCAT）,用 Python 侧批量或方言兼容写法。 |
| 租户边界 | 仅限同租户空间互移;服务端对跨租户目标复用现有 18041 拒绝。 |
| 撤回边界 | 同空间撤回可能失败（原位置被占/原目录被删）,要有失败提示。 |
