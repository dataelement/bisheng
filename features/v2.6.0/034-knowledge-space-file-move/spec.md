# Feature: 知识空间文件 / 文件夹移动（同空间 + 跨空间）+ 文件夹上传

> **本文档定位 — 纯 What（需求口径，不随代码漂移）**
> spec 只回答 做什么 / 验收标准 / 不做什么。所有 How（数据流、字段、API、OpenFGA 模型、向量迁移管线、前端组件、错误码定义）写在 [design.md](./design.md) 与 [tasks.md](./tasks.md)。

**关联 PRD**: 2.6 PRD §5.2 知识空间适配支持文件夹和文件移动 + §5.5 知识空间内上传文件夹（均做到 beta4，中粮升级为 beta4）
**优先级**: P1
**所属版本**: v2.6.0
**依赖**: F004 / F008（ReBAC core / resource-rebac-adaptation）；F027（列表 cursor 分页 INV-6）；F039（文档版本管理——跨空间移动需整链迁移）

> **范围边界**
> - **本次纳入**：
>   - 同一知识空间内，文件 / 文件夹的移动，两种交互：「移动到」弹窗 + 多选拖拽。
>   - **跨知识空间移动**（文件与文件夹都支持）：元数据即时迁移 + 检索数据异步迁移，迁移期间文件显示「处理中」。
>   - 新增 `move_folder` / `move_file` 两个关系类型权限。
>   - 移动的循环 / 层级 / 重名（仅同空间）/ 权限校验，批量部分失败处理，同空间移动成功后的「撤回」。
>   - **文件夹上传**（§5.5）：拖拽 + 「新增」两入口上传文件夹，**全量嵌套**（子文件夹内所有文件按目录树重建一并上传）；单次批量≤1000 文件、层级≤10 级、文件夹重名、容量上限等上传校验；不支持格式 / 隐藏 / 超大文件静默过滤。
> - **本次明确排除**（PRD 已拍板的跨空间简化）：
>   - 跨空间移动**不做重名校验**。
>   - 跨空间移动**不迁移知识空间标签**——移动完成后清空文件原有的空间标签（AC-23）。
>   - 跨空间移动**不做撤回**——以移动前的二次确认替代。
>   - 移动过程不重新解析文档内容。
>   - **服务端不对目标空间 / 文件夹重复校验「上传文件」权限**——目标侧权限过滤仅为「移动到」弹窗的展示行为（产品 2026-06-10 拍板）。

---

## 1. 用户故事

作为 **知识空间的成员（拥有移动权限）**，
我希望 **能把文件和文件夹移动到本空间内的其它目录、或我有上传权限的其它知识空间（点选「移动到」或直接拖拽，支持多选一起移）**，
以便 **跨空间整理资料归属，而不必下载重传。**

作为 **空间管理员**，
我希望 **移动权限能按关系档位（所有 / 管理 / 编辑 / 查看）默认配置**，
以便 **只让有编辑及以上权限的人移动内容，查看者不能动。**

作为 **有上传权限的知识空间成员**，
我希望 **把本地整个文件夹（含子文件夹）一次拖拽或新增上传到知识空间，系统自动按原目录结构重建目录树**，
以便 **批量导入已有资料而不必逐个建文件夹再上传。**

## 2. 验收标准

### 2.1 移动入口与交互

- **AC-01** — WHEN 用户对有移动权限的文件 / 文件夹选择「移动到」并选定目标（本空间内某文件夹 / 空间根，或其它空间及其文件夹）, THE SYSTEM SHALL 把对象移动到目标下，列表刷新反映新位置。
- **AC-02** — THE SYSTEM SHALL 支持多选后「批量移动到」与「批量拖拽移动」。
- **AC-03** — WHEN 用户拖拽选中项到当前页面可视范围内的目标并松手, THE SYSTEM SHALL 执行移动。可投放目标包括：本空间列表中的**文件夹行**（同空间移动），以及左侧空间列表中的**其它知识空间**（跨空间移动，落到该空间根目录，松手后按 AC-18 二次确认）。WHILE 拖拽进行中, THE SYSTEM SHALL NOT 展开文件夹内部或跳转进入其它空间（只能落在当前可视目标上）。
- **AC-04** — 「移动到」弹窗 SHALL 左侧展示知识空间列表，右侧展示选中空间内的文件夹与文件。

### 2.2 权限

- **AC-05** — THE SYSTEM SHALL 要求：移动文件需用户对该文件有 `move_file` 权限；移动文件夹需对该文件夹有 `move_folder` 权限。
- **AC-06** — THE SYSTEM SHALL 在关系类型中新增 `move_folder`（移动文件夹）与 `move_file`（移动文件）；「所有 / 管理 / 编辑」三档默认勾选，「查看」默认不勾选。
- **AC-07** — 「移动到」弹窗 SHALL 只把用户有「上传文件」权限的知识空间与文件夹作为可选目标；文件仅在用户有查看权限时展示，且一律置灰不可选（文件不可作为移动目标）。
- **AC-08** — THE SYSTEM SHALL 仅校验被移动对象本身的移动权限，不校验其子文件 / 子文件夹的权限。
- **AC-09** — THE SYSTEM SHALL 保证：移动前后，对象通过成员管理直接授予的权限不变；移动后按新位置重新计算其继承权限。

### 2.3 移动校验

- **AC-10** — IF 目标文件夹是被移动文件夹自身、其子目录、或其当前所在的父目录, THEN THE SYSTEM SHALL 拒绝该项移动并提示。
- **AC-11** — IF 移动后被移动子树的最深层级超过现有 10 层限制, THEN THE SYSTEM SHALL 拒绝该项移动并提示。
- **AC-12** — IF **同空间**移动的目标目录内已存在同名文件 / 文件夹, THEN THE SYSTEM SHALL 拒绝该项移动并提示「存在同名文件 / 文件夹」；跨空间移动 SHALL NOT 做重名校验。
- **AC-13** — THE SYSTEM SHALL 允许移动处于「成功 / 排队中 / 解析中 / 失败 / 违规 / 超时 / 重建中」任意状态的文件。

### 2.4 批量与异常处理

- **AC-14** — WHEN 批量移动（点确定或拖拽松手）的勾选项中存在无权限移动的项, THE SYSTEM SHALL 弹窗提示「存在无权限移动的文件 / 文件夹：{名称}」，并提供两个操作：【移动其余文件】【取消移动】。
- **AC-15** — WHEN 批量移动中存在其它异常项（移动到子目录 / 超过深度限制 / 同空间重名等), THE SYSTEM SHALL 同样弹窗列出问题项，并提供继续移动其余 / 取消的选择。

### 2.5 同空间移动的撤回

- **AC-16** — WHEN 同空间移动成功, THE SYSTEM SHALL 弹出「移动成功」toast，且 toast 内含「撤回」入口。
- **AC-17** — WHILE 该成功 toast 尚未消失, IF 用户点击「撤回」, THEN THE SYSTEM SHALL 把本次移动的对象移回原位置。

### 2.6 跨空间移动

- **AC-18** — WHEN 移动目标位于另一知识空间, THE SYSTEM SHALL 在执行前弹出二次确认；确认后执行并 toast 提示移动成功（**不含撤回入口**）。
- **AC-19** — WHEN 跨空间移动确认后, THE SYSTEM SHALL 立即把对象（含文件夹整棵子树）呈现在目标空间的目标位置；其检索数据异步迁移，迁移期间相关文件状态显示「处理中」，迁移完成后恢复正常并可在目标空间被检索。
- **AC-20** — THE SYSTEM SHALL 把文件的主版本与全部历史版本一起移动到新位置，保持原有版本管理关系。
- **AC-21** — WHEN 处于「排队中 / 解析中」的文件被跨空间移动, THE SYSTEM SHALL 让其解析在目标空间继续完成（解析产物落入目标空间的检索库）。
- **AC-22** — IF 跨空间移动的检索数据迁移失败, THEN THE SYSTEM SHALL 把该文件标记为可重试的失败态（不丢失文件本体与版本关系），并允许重试。
- **AC-23** — WHEN 跨空间移动完成, THE SYSTEM SHALL 清空被移动文件原有的知识空间标签（不迁移到目标空间）。

### 2.7 文件夹上传（PRD §5.5）

- **AC-24** — THE SYSTEM SHALL 支持两个入口上传文件夹：① 把文件夹拖拽到知识空间列表区；② 右上角「新增」中选择上传文件夹。两个入口均支持**嵌套**——文件夹内所有子文件夹中的文件都按原目录结构一并上传。
- **AC-25** — THE SYSTEM SHALL 由前端读取所上传文件夹内每个文件自带的相对目录结构，后端据此在目标位置**重建目录树**，文件落入对应层级后继续走既有的解析 / 检索流程。
- **AC-26** — THE SYSTEM SHALL 以**前端读取到的原始文件总数**（含所有嵌套子文件夹、过滤之前）为准限制单次批量上传 ≤ 1000 个文件；IF 超过 1000, THEN THE SYSTEM SHALL 提示「单次批量文件总数最多 1000 个」并整批不上传。
- **AC-27** — WHEN 上传文件夹, THE SYSTEM SHALL **静默过滤**以下文件（不报错、不进入上传流程、不占用合规文件的上传名额），其余合规文件正常上传：① 平台不支持的格式；② 隐藏文件；③ 超过单文件大小上限（默认 200MB）的文件。
- **AC-28** — IF 上传后在目标位置生成的文件夹最深层级超过 10 级, THEN THE SYSTEM SHALL 拒绝该文件夹上传并报错。
- **AC-29** — IF 当前目录下已存在与待上传文件夹同名的文件夹, THEN THE SYSTEM SHALL 拒绝上传该文件夹并提示「该位置已存在同名文件夹」。
- **AC-30** — IF 上传后将超出用户 / 角色的知识空间文件容量上限，或超出企业 / 租户的总存储容量上限, THEN THE SYSTEM SHALL 拒绝上传并报错。
- **AC-31** — IF 上传的文件在知识空间内已存在同名文件, THEN THE SYSTEM SHALL 按现有重复文件逻辑处理（复用既有行为，本需求不重定义）。
- **AC-32** — 文件夹上传被拒绝的所有场景（数量超 1000 / 层级超 10 / 文件夹重名 / 容量超限）, THE SYSTEM SHALL 以 **toast 提示**告知用户具体拒绝原因（产品 2026-06-11 补充）。

## 3. 边界情况

- **撤回失败（同空间）**：撤回时若原位置已被同名对象占用、或原父目录已被删除，系统应提示撤回失败，且不破坏当前（移动后）状态。
- **无效拖拽**：拖到对象自身所在行、或拖到其当前所在目录，视为无效移动，不产生变化。
- **文件夹级联**：移动文件夹时，其全部子孙的层级路径需随之更新；跨空间时子树内全部文件（含各自历史版本）逐个走检索数据迁移。对外可观测行为是移动后整棵子树出现在新位置。
- **部分成功**：批量移动允许部分项成功、部分项被拒（按 AC-14/15 用户选择「移动其余」后），返回需让前端能区分每项结果。
- **并发**：同空间移动与他人同时在目标目录创建同名对象竞态时，以服务端重名校验（AC-12）为准。
- **跨空间检索窗口**：迁移完成前，文件在目标空间列表可见但内容暂不可检索（AC-19 已声明），属预期行为。
- **空间检索模型不同**：源 / 目标空间使用不同向量模型时，迁移自动适配，对用户透明（仅迁移时长不同）。
- **上传后全部被过滤（§5.5）**：若一个文件夹内的文件经 AC-27 过滤后无任何合规文件，则没有可上传内容，系统提示无可上传文件，不创建空目录树。
- **上传校验分工（§5.5）**：文件总数 / 格式 / 单文件大小可在前端读取目录结构时即时判定；文件夹层级、文件夹重名、容量上限以**服务端校验为准**（与他人并发上传 / 建夹竞态时同理）。具体校验落点写在 design.md。
