## 1. 组织层级（部门树）

| 层级 | 说明 |
|------|------|
| L1 | **星辰公司**（根 / 虚拟公司） |
| L2 | **测试部门**（父级：星辰公司） |
| L3 | **测试A**、**测试B**（父级：测试部门） |

## 2. 人员与角色（按部门）

| 父级部门 | 部门名称 | 人员名称 | 角色 |
|----------|----------|----------|------|
| 星辰公司 | 测试部门 | 测试领导 | 部门管理员 |
| 星辰公司 | 测试部门 | 测试用户 | 测试角色（可创建） |
| 测试部门 | 测试A | 测试A领导 | 部门管理员 |
| 测试部门 | 测试A | 测试A用户 | 测试角色（可创建） |
| 测试部门 | 测试B | 测试B领导 | 部门管理员 |
| 测试部门 | 测试B | 测试B用户 | 测试角色（**不可创建**） |

### 测试意图速记

- **部门管理员**：各层「领导」账号，用于验证部门管理范围、子部门用户可见性等。
- **可创建 vs 不可创建**：`测试用户`、`测试A用户` 使用「测试角色（可创建）」；**`测试B用户`** 使用「测试角色（**不可创建**）」——用作 **`create_app` / 创建类能力** 的**负例**。
- 两条「可创建」与一条「不可创建」并列，便于对照 UI（如创建应用、复制应用与创建权限对齐）与后端校验。

## 3. 自定义角色与作用域（角色表）

以下两条为系统中配置的**角色名称**及其**作用域**（与上表中的「测试角色（可创建/不可创建）」对应）：

| 角色名称 | 作用域 |
|----------|--------|
| 测试角色（不可创建） | 虚拟公司：星辰公司 / 测试部门 |
| 测试角色（可创建） | 虚拟公司：星辰公司 / 测试部门 |

两条角色**作用域相同**（均在「星辰公司 / 测试部门」下），差异在角色本身是否具备「可创建」类权限，适合做对照实验。
