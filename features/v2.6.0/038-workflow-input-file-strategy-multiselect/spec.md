# Feature: 工作流输入节点文件处理策略（单选 + 输出变量联动）

**关联 PRD**: [2.6 release · §工作流输入节点文件处理策略支持多选需求](https://dataelem.feishu.cn/wiki/NEtywYou0iJYfhka6RwcXkGAnpd)
**优先级**: P1
**所属版本**: v2.6.0
**依赖**: 无

> **⚠️ 需求变更（2026-06-26，按 06.24 评审更新版 PRD 重写）**
> PRD 标题仍叫「支持多选」，但更新后的正文明确：对话框与表单的「文件处理策略」**保持原有单选逻辑**。
> 所谓「多选」指的是——**选定某一个策略时，按上传文件类型联动暴露多个输出变量**，而非让用户勾选多个策略。
> 上一版（真·多选：对话框按文档/图片分组、表单 checkbox 多选）已作废，本文档为重写版。差异见 §5。

> **范围边界**
> - **本次纳入**：
>   - **对话框输入**：文件处理策略**单选**（解析文件内容 / 不解析），选定后按上传文件类型（全部/文档/图片）联动展示输出变量。
>   - **表单输入**：文件处理策略**单选**，且把选项重命名为三项——「解析文件内容(不存入临时知识库)」「解析文件内容(存入临时知识库)」「不解析(原始文件)」；选定后按上传文件类型联动展示输出变量与配置项。
>   - 输出变量按**统一规则**联动（见 §3）：路径恒有、图片按类型、解析结果按是否解析、临时库名按是否入库。
>   - 后端变量暴露规则改为「按统一规则产出并集」；旧字符串 / 数组形态向后兼容。
> - **本次明确排除**：
>   - 历史工作流数据迁移（旧值运行时自动兼容，不批量改库）。
>   - 新增文件处理模式（底层仍是 extract_text / keep_raw / ingest_to_temp_kb 三种原子模式）。
>   - 真·多选 UI（用户自由勾选多个策略）——已被本次「单选 + 命名好的组合项」方案取代。

---

## 1. 用户故事

作为 **工作流搭建者**，
我希望 **选一个文件处理策略后，能一次拿到该场景下所有有用的输出变量（解析文本、原图、文件路径、临时知识库），并随上传的文件类型自动适配**，
以便 **不用在「只要文本」「只要原图」之间纠结，也不用为「既解析又入库」额外配置——常用组合直接给成命名选项。**

---

## 2. 验收标准

### 2.1 对话框输入（策略单选，2 选项）

策略选项：**解析文件内容**（extract_text）/ **不解析(原始文件)**（keep_raw）。

| ID | 上传文件类型 | 策略 | 应展示的输出变量 |
|----|------|------|---------|
| AC-01 | 全部类型 | 解析文件内容 | 解析结果长度上限 + `dialog_files_content` + `dialog_image_files` + `dialog_file_paths` |
| AC-02 | 文档 | 解析文件内容 | 解析结果长度上限 + `dialog_files_content` + `dialog_file_paths` |
| AC-03 | 图片 | 解析文件内容 | 解析结果长度上限 + `dialog_files_content` + `dialog_image_files` + `dialog_file_paths` |
| AC-04 | 全部类型 | 不解析 | `dialog_image_files` + `dialog_file_paths` |
| AC-05 | 文档 | 不解析 | `dialog_file_paths` |
| AC-06 | 图片 | 不解析 | `dialog_image_files` + `dialog_file_paths` |

### 2.2 表单输入（策略单选，3 选项 / 重命名）

策略选项：**解析文件内容(不存入临时知识库)**（extract_text）/ **解析文件内容(存入临时知识库)**（extract_text + ingest_to_temp_kb）/ **不解析(原始文件)**（keep_raw）。

| ID | 上传文件类型 | 策略 | 应展示的输出变量 / 配置项 |
|----|------|------|---------|
| AC-07 | 全部类型 | 解析(不入库) | 长度上限 + 解析结果变量名 + 图片变量名 + 文件路径变量名 |
| AC-08 | 文档 | 解析(不入库) | 长度上限 + 解析结果变量名 + 文件路径变量名 |
| AC-09 | 图片 | 解析(不入库) | 长度上限 + 解析结果变量名 + 图片变量名 + 文件路径变量名 |
| AC-10 | 全部类型 | 解析(入库) | 长度上限 + 解析结果变量名 + 图片变量名 + 文件路径变量名 + 临时知识库名 |
| AC-11 | 文档 | 解析(入库) | 长度上限 + 解析结果变量名 + 文件路径变量名 + 临时知识库名 |
| AC-12 | 图片 | 解析(入库) | 长度上限 + 解析结果变量名 + 图片变量名 + 文件路径变量名 + 临时知识库名 |
| AC-13 | 全部/图片 | 不解析 | 图片变量名 + 文件路径变量名 |
| AC-14 | 文档 | 不解析 | 文件路径变量名 |

### 2.3 数据结构与执行

- **AC-15** — 对话框策略以**单字符串**存（`extract_text` / `keep_raw`）；表单策略以**数组**存：解析(不入库)=`[extract_text]`、解析(入库)=`[extract_text, ingest_to_temp_kb]`、不解析=`[keep_raw]`。
- **AC-16** — WHERE 读到历史值（旧单字符串、或上一版的 map/数组），THE SYSTEM SHALL 归一化后正确执行，旧工作流不报错。
- **AC-17** — WHEN 执行，THE SYSTEM SHALL 一次解析、按**统一规则**产出变量（见 §3），不重复解析、不重复入库。
- **AC-18** — 下游节点变量选择器 SHALL 只列出当前「策略 × 上传类型」实际产出的变量。

## 3. 统一变量暴露规则（核心）

对话框与表单**同一套规则**（命名不同：对话框 `dialog_*`，表单为用户自定义变量名）：

| 输出 | 暴露条件 |
|---|---|
| 文件路径（path） | **恒暴露**（原始文件始终保留） |
| 图片（image） | 上传文件类型 ∈ {全部, 图片} |
| 解析结果（content）+ 解析长度上限 | 策略含「解析」（解析 / 解析入库） |
| 临时知识库名（key） | 策略 = 解析(存入临时知识库)（仅表单） |

> 与旧逻辑最大区别：**「解析」也会暴露 path + image**（旧版 extract 只给 content）；图片是否暴露**只看上传类型、不再看是否 keep_raw**。

---

## 4. 边界情况

- 上传类型在「全部 ↔ 文档 ↔ 图片」切换时，图片变量随之显隐；隐藏的变量不应残留在下游可选列表。
- 解析文本总量仍受 `file_content_size`（默认 15000 字）上限；多产出下解析只做一次、产物复用。
- 历史值可能是字符串、也可能（上一版升级后）是 map / 数组 —— 读取侧三种都兼容。
- 表单旧「存入临时知识库」(纯 ingest) → 新「解析(入库)」(extract+ingest)：见 design §4.4 迁移说明。

## 5. 与上一版（真·多选）的差异

| | 上一版（已作废 / 已部署 120 待回退） | 本版（新 PRD） |
|---|---|---|
| 对话框 | 按文档/图片**分组、组内单选**、存 `{doc,image}` map | **一个单选下拉**（2 项）、存单字符串 |
| 表单 | **checkbox 多选**、keep_raw 互斥、存数组 | **单选下拉 3 项**（重命名）、存固定数组 |
| 变量暴露 | 按所选模式精确暴露（extract→只 content） | 按**统一规则**暴露并集（解析→content+path+image） |
| 图片变量 | 仅图片组 keep_raw 时 | 上传类型∈{全部,图片} 即暴露 |
