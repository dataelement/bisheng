# BiSheng 项目数据库表结构与关联说明

> 说明：本文档基于项目源码中的 SQLModel/ORM 模型生成，覆盖当前仓库可识别的 65 张业务表。中文释义以工程语义翻译为主；“显式外键”表示数据库层有 `ForeignKey` 声明，“约定型关联”表示代码/字段命名上存在明确业务关联，但数据库层未必强约束。

## 1. 建议使用的 SQL

### 1.1 查询全部表名
```sql
SELECT table_name AS table_name
FROM information_schema.tables
WHERE table_schema = DATABASE()
ORDER BY table_name;
```

### 1.2 查询全部字段 schema
```sql
SELECT
  c.table_name,
  c.ordinal_position,
  c.column_name,
  c.column_type,
  c.is_nullable,
  c.column_key,
  c.column_default,
  c.extra,
  c.column_comment
FROM information_schema.columns c
WHERE c.table_schema = DATABASE()
ORDER BY c.table_name, c.ordinal_position;
```

### 1.3 查询主键 / 唯一键 / 外键
```sql
SELECT
  tc.table_name,
  tc.constraint_name,
  tc.constraint_type,
  kcu.column_name,
  kcu.referenced_table_name,
  kcu.referenced_column_name
FROM information_schema.table_constraints tc
LEFT JOIN information_schema.key_column_usage kcu
  ON tc.constraint_schema = kcu.constraint_schema
 AND tc.table_name = kcu.table_name
 AND tc.constraint_name = kcu.constraint_name
WHERE tc.table_schema = DATABASE()
ORDER BY tc.table_name, tc.constraint_type, tc.constraint_name;
```

## 4. 字段关联规则（读表时优先关注）

- `tenant_id`：几乎所有核心业务表都带租户隔离字段，逻辑上关联 `tenant.id`。
- `user_id` / `applicant_user_id` / `updater_id` / `create_user`：逻辑上关联 `user.user_id`。
- `department_id` / `root_dept_id` / `parent_id`：分别表示部门归属、租户根部门、同表树形父节点。
- `knowledge_id` / `space_id`：逻辑上关联知识库空间 `knowledge.id`。
- `server_id` / `model_id`：逻辑上关联 LLM 供应商与模型表。
- `session_id`：在聊天与灵思模块里通常关联 `message_session.chat_id`。
- `flow.id`、`assistant.id`、`share_link.flow_id` 一类字段承担应用、流程、分享之间的串联。
