# 一、背景

1. **<span style="color: inherit; background-color: rgba(205,178,250,0.7)">需求整合与规划</span>**<span style="color: inherit; background-color: rgba(205,178,250,0.7)"> 源于中粮定制需求，因具备通用价值纳入主版本。本期聚焦</span>**<span style="color: inherit; background-color: rgba(205,178,250,0.7)">基础功能闭环</span>**<span style="color: inherit; background-color: rgba(205,178,250,0.7)">，上半年将持续迭代翻译及 AI 高阶能力。</span>

2. **&#x20;核心功能定义** 借鉴客户内部高认可度的成熟竞品，降低用户学习门槛：

   1. **订阅（外部情报窗口）**：对标“语鲸”。聚合外部差量信息，满足资讯与政策的实时追踪需求。

   2. **知识空间（知识复利空间）**：对标“腾讯 IMA”。打造个人及团队专属 AI 知识库，沉淀长期知识资产。

3. **关联系统改造** 配合上述功能落地，同步升级**前台问答、站内信、后台权限及底层模型系统**等基础设施。

<li><strong>文件过滤</strong>：在文件管理窗口中，系统需自动过滤，将<strong>不支持</strong>的文件类型进行<strong>置灰</strong>或<strong>隐藏</strong>处理，用户无法选中。</li>

<li><strong>图片类</strong>：png、jpg、jpeg、bmp</li>
</ol>
</li>
<li>
<p><strong>大小限制</strong>：单个文件最大支持 <strong>200MB</strong>。</p>
</li>
<li>
<p><strong>数量限制</strong>：单次批量上传最多支持 <strong>50</strong> 个文件。</p>
<ol>
<li><em>异常提示</em>：若用户选择超过 50 个文件，阻止上传并弹出 Toast 提示：“单次最多允许上传 50 个文件”。</li>
<li><em>异常提示</em>：若选中文件中包含超大文件，阻止该文件上传并提示：“文件 [文件名] 超过 200MB 限制”。</li>
</ol>
</li>
</ol>
</li>

## 1. 支持第三方OCR 接入`@唐义芳`

1. 需求背景及目标

   1. 需求背景：中粮要求支持除 ETL 以外的文档解析服务，实现自主可控

   2. 需求目标

      1. 通过系统配置选择当前生效的文档解析服务

      2. 在文档解析服务实现切换后，知识库解析功能仍然可以正常使用

2. 需求描述

   1. 系统配置中，支持配置 Paddle OCR/MinerU 解析服务。

   2. 配置 Paddle OCR/MinerU 解析服务后， 平台知识库PDF/图片解析将调用对应的解析服务。

   ```yaml
   knowledges:
       etl4lm:
           # provider:"bisheng_etl"
           # url: "http://129.226.64.9:50103/v1/etl4llm/predict"  
           # timeout: 3600
           # ocr_sdk_url: "http://192.168.106.12:8502"# http://192.168.106.12:8502
           
           provider:"MinerU"
           # minerU 相关配置
           
           # provider:"Paddle OCR"
           # Paddle OCR 相关配置

   ```

## 5. 是否应该允许将频道加入知识库

1. 结论：无法将频道一起加入，只能加入单篇文章

2. 理由：

   1. 功能定位：订阅是资讯的汇集，主要强调即时性，知识是有价值内容的沉淀，只收集用户已经阅读过认为有价值的内容

   2. 资源管理：如果允许用户将频道都加入知识库，可能会出现大量重复知识库以及知识库中大量冗余信息的问题
