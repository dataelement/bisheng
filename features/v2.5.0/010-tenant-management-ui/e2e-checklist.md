# E2E 验证清单: F010 Tenant Management UI

**测试环境**: http://192.168.106.114:4001 (Platform)
**前置条件**: 
- 后端 multi_tenant.enabled = true
- 至少一个管理员账号 (admin/admin123)
- 至少两个普通用户（用于租户用户管理测试）

---

## Platform 前端

### AC-7.1: 租户管理菜单可见性
- [ ] 以管理员 (admin) 登录 Platform
- [ ] 侧边栏出现「租户管理」菜单项（仅 multi_tenant_enabled=true 时）
- [ ] 退出，以普通用户登录
- [ ] 侧边栏不显示「租户管理」菜单项
- [ ] 修改后端 multi_tenant.enabled=false，管理员登录后也不显示

### AC-7.2: 租户列表页
- [ ] 管理员点击「租户管理」进入列表页
- [ ] 表格显示：名称、编码、状态(彩色badge)、用户数、存储用量、创建时间
- [ ] 输入搜索关键字，列表按名称/编码过滤
- [ ] 分页正常：切换页码，数据正确加载
- [ ] 点击用户数 → 弹出用户管理 Dialog

### AC-7.3: 创建租户
- [ ] 点击「创建租户」按钮
- [ ] Dialog 显示：名称(必填)、编码(必填)、Logo、联系人、管理员选择(必填)
- [ ] 编码校验：输入 `1bad` → 提示格式错误
- [ ] 编码校验：输入 `good_code` → 校验通过
- [ ] 管理员选择：搜索用户 → 出现下拉列表 → 点击添加 → 显示为 tag
- [ ] 提交 → 列表刷新，新租户出现
- [ ] 编辑已有租户 → 编码字段 disabled + 提示"不可修改"

### AC-7.4: 停用租户
- [ ] 列表中点击「停用」按钮
- [ ] 弹出二次确认对话框
- [ ] 警告文案包含"强制下线"提示
- [ ] 确认后状态变为「已停用」(灰色 badge)

### AC-7.5: 删除租户
- [ ] 对已停用租户，列表显示「删除」按钮（正常状态不显示）
- [ ] 点击删除 → 弹出二次确认 Dialog
- [ ] 需输入租户编码才能启用确认按钮
- [ ] 有活跃用户时：后端返回错误提示
- [ ] 移除所有用户后：删除成功，列表刷新

### AC-5.5: 登录租户选择页
- [ ] 创建一个用户属于两个租户
- [ ] 该用户登录 → 跳转到 /tenant-select 页面
- [ ] 页面显示两个租户卡片（Logo/首字母 + 名称 + 编码）
- [ ] 卡片按 last_access_time 降序排列
- [ ] 点击一个卡片 → 跳转主页
- [ ] 点击「返回登录」 → 回到登录页

### AC-5.6: 单租户自动进入
- [ ] 只属于一个租户的用户登录 → 直接进入主页（不显示选择页）

### AC-6.4: Header 租户切换器
- [ ] multi_tenant_enabled=true 时，Header 右侧显示当前租户名 + 下拉箭头
- [ ] 悬停/点击 → 下拉显示其他租户列表
- [ ] 当前租户带 ✓ 标记
- [ ] 管理员用户：下拉底部显示「租户管理」入口
- [ ] 普通用户：不显示「租户管理」入口

### AC-6.5: 切换租户
- [ ] 在下拉中选择另一个租户
- [ ] 弹出确认对话框"切换企业将刷新页面..."
- [ ] 确认后页面刷新
- [ ] Header 显示新租户名
- [ ] 数据加载为新租户上下文（如知识库列表不同）
