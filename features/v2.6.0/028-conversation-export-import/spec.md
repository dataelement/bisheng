> **范围边界**
> - **本次纳入**：
>   - 工作台「日常模式」、「应用-助手」、「应用-工作流」三种会话场景的**回答级**导出（Word / PDF / Markdown / TXT）。
>   - 同一选择态下「导入到知识空间」能力，目标为知识空间根目录或文件夹，复用现有上传 + 解析 + 入库链路。
>   - PC 与 H5 两端 UI；H5 走底部弹层。
>   - 列出当前用户可上传的知识空间 / 文件夹的能力（作为导入目标选择器的数据源）。
> - **本次明确排除**：
>   - 整会话导出（顶部 `/api/v1/audit/session/export/data` 维持原状，仅返 JSON，归审计用途）。
>   - Linsight、Channel Article 等非工作台会话的回答导出。
>   - 流式生成中边导边停 / 中途取消 / 暂存草稿。
>   - 自动云存归档：本期不把生成的文件落仓库，请求结束即清理临时文件。
>   - 思考过程 / 工具调用过程 / RAG 命中片段的导出（只导最终答案文本）。
>   - 覆盖同名文件、人工选择覆盖 vs 重命名（PRD 已定「自动追加序号」，本期不开放选项）。
>   - 导入文件大小校验：聊天记录生成的 md 体量恒定较小，本接口不做单文件大小限制；知识空间整体配额仍按现有上传链路校验（AC-23）。
>   - 导入侧敏感词二次过滤：会话内容生成阶段已过滤；现有上传链路若仍触发，按其原有 `sensitive_status` 展示，不阻断导入响应。

## 1. 用户故事

> 只保留业务价值视角。研发视角的「为什么这么实现」（统一选择态、共用 Service）属于设计意图，
> 见 design.md §3 决策 + §4.3 模块职责，不在 spec 重复。

**故事 A（业务用户 · 工作台日常模式）**：
作为 **使用工作台日常模式与模型对话的用户**，
我希望 **把某条 AI 回答连同我的提问，直接导出成 Word / PDF / Markdown / TXT 拿走**，
以便 **离线归档、转交同事或粘进项目报告，不必手动复制再排版**。

**故事 B（业务用户 · 应用会话）**：
作为 **在工作台打开助手 / 工作流应用使用的用户**，
我希望 **把应用产出的回答按回答粒度沉淀下来**，
以便 **复用助手回答的结论，而不是只能保留整段会话历史**。

**故事 C（知识沉淀者 · 中信保 / 首钢场景）**：
作为 **希望把高质量 AI 回答沉淀为企业知识资产的用户**，
我希望 **在选择回答后一键导入到某个知识空间 / 文件夹，自动走入库解析**，
以便 **未来在知识空间检索时能命中这些经过验证的回答，形成可复用的企业知识**。

## 2. 验收标准

> spec 的核心资产。AC-ID 在本特性内唯一，格式 `AC-NN`。
> tasks.md 测试任务必须通过 `覆盖 AC: AC-NN` 追溯到此表。
> 错误码号在此表中仅作为「可观测的对外行为」引用；错误码的定义与归属是实现细节，见 design §6 / 代码 `common/errcode/workstation.py`。

### 2.1 导出能力

| ID | 角色 | 操作 | 预期结果 |
|----|------|------|---------|
| AC-01 | 任意已登录用户 | 在一条**已完成生成**的 AI 回答下方点击「导出」icon | 当前会话进入消息选择态；本回答与其对应 query 默认勾选；底部出现固定操作栏（PC：全选 + 4 格式按钮 + 导入到知识空间 + 关闭；H5：全选 + 导出到本地 + 导入到知识空间 + 关闭） |
| AC-02 | 用户 | 选择态下点击「导出为 Word / PDF / Markdown / TXT」 | 后端同步返回对应格式文件流；浏览器按 `Content-Disposition` 文件名下载；下载完成后选择态保持，不自动退出 |
| AC-03 | 用户 | 勾选某条 answer | 对应 query 自动联选；反向亦然；取消勾选同步取消 |
| AC-04 | 用户 | 点击「全选」 | 当前会话所有 message 进入选中态；上滑加载更多历史时，新加载消息也自动选中 |
| AC-05 | 用户 | 在某条消息上点击「全选以下消息」 | 该消息（含自身）及下方所有 message 进入选中态 |
| AC-06 | 用户 | 对同一 query「重新生成」产生多条 answer，勾选任一 | 该 query 与**所有** answer 均被勾选；导出文件按时间顺序依次包含全部 answer（每条独立成段，前置标签 `<模型名/应用名>：`） |
| AC-07 | 用户 | AI 回答正流式生成中 | 该 answer 下方不展示「导出」icon；与现有「复制」icon 可见性条件完全一致 |
| AC-08 | 用户 | 选中超过 200 条并点击任意导出 / 导入 | 前端 toast「单次最多选中 200 条消息」并阻止请求；后端同样做 200 条上限校验，超过返错误码 12061 |
| AC-09 | 用户 | 工作流 OUTPUT 节点输出含表单 / 按钮 / 附件等非纯文本 | 导出文件对应位置渲染为 `[交互组件：<节点类型>]` 占位文本；不报错、不丢上下文 |
| AC-10 | 用户 | 查看导出文件名 | 格式 `<会话标题>_yyyyMMddHHmm.<ext>`；标题为空用 `未命名会话`；截断到 80 字符；按 Windows 最严标准把非法字符、控制字符、末尾点与空格替换 / 删除为 `_` |
| AC-11 | 用户 | 打开 Word / PDF 导出文件 | A4 纵向白底；正文中文字体；用户名段、模型名段单独成段且加粗 14pt；query 与 answer 之间一条灰色横线；编号列表 / 表格 / 代码块 / 图片正确渲染 |
| AC-12 | 用户 | 打开 Markdown 导出文件 | 保留 Markdown 结构；用户名段 `**Admin：**`、模型名段 `**DeepSeek v3.2：**`，每对 query/answer 间一条 `---`；图片用 `![](url)`，URL 保持内部地址（PRD 决策，外网失效不补救） |
| AC-13 | 用户 | 打开 TXT 导出文件 | 纯文本；保留段落换行；表格降级为 Tab 对齐；图片以 `[图片：<文件名>]` 占位；代码块保留缩进；各段用空行分隔 |
| AC-14 | 用户 | 导出含 RAG 引用角标的回答 | 所有现有形式的角标被完全剥除；不出现「跳转链接」、不在末尾追加「来源列表」 |
| AC-15 | 用户 | 导出 `agent_answer` 类回答 | 文件只含最终文本；不含 `thinking` / `tool_call` / `tool_result` / 中间步骤；不报错、不留空段 |

### 2.2 导入知识空间能力

| ID | 角色 | 操作 | 预期结果 |
|----|------|------|---------|
| AC-16 | 用户 | 选择态下点击「导入到知识空间」 | 弹出知识空间选择弹窗；列表数据源为「当前用户可上传的知识空间」 |
| AC-17 | 用户 | 查看可选知识空间列表 | 列出当前用户持有 `upload_file` 权限的全部知识空间；支持搜索；支持展开文件夹；支持选「根目录」或「具体文件夹」 |
| AC-18 | 用户 | 选好目标点击确认 | 后端同步返回导入结果；前端 toast「导入成功，文件正在解析中」附知识空间链接；选择态保持 |
| AC-19 | 用户 | 目标文件夹已存在同名文件 | 后端追加 `(1)`、`(2)`…直到不重名再入库；响应标记已重命名 + 最终落地文件名 |
| AC-20 | 用户 | 对目标空间 / 文件夹无 `upload_file` 权限 | 返错误码 12067；前端 toast「暂无权限导入到该知识空间」 |
| AC-21 | 用户 | 目标空间不存在 / 已删除 | 返错误码 12065；前端 toast「知识空间不存在或已删除」 |
| AC-22 | 用户 | 目标文件夹不存在 / 已删除 | 返错误码 12066；前端 toast「文件夹不存在或已删除」 |
| AC-23 | 用户 | 已用满知识空间文件容量 | 复用现有配额校验链路，归一化映射到错误码 12068；前端按现有「配额已满」文案 |
| AC-24 | 用户 | 入库后立即返回 | 文件状态走现有链路 `WAITING → PROCESSING → SUCCESS / FAILED / TIMEOUT`；本接口不等待解析完成；前端在知识空间页面看进度 |

### 2.3 权限与越权边界

| ID | 角色 | 操作 | 预期结果 |
|----|------|------|---------|
| AC-25 | 用户 A | 携带用户 B 的 message_ids 调导出 / 导入 | 后端按 `(message_id, user_id)` 过滤，命中 0 条 → 返错误码 12062；前端 toast「消息不存在或无访问权」 |
| AC-26 | 用户 A | message 隶属其他租户的会话 | 多租户事件自动注入 tenant 过滤，命中 0 条 → 同 AC-25 错误码；不暴露「跨租户」语义 |
| AC-27 | 用户 | message_ids 混入不存在 / 已删除 ID | 整批拒绝，返错误码 12060；不做「部分导出」妥协；前端 toast「消息不存在或已删除，请刷新会话」 |
| AC-28 | 用户 | 选中 message_ids 跨多个会话 | 后端校验全部 message 必须同会话；不同会话返错误码 12062；前端选择态本身限制在单会话内 |

### 2.4 渲染与失败处理

| ID | 角色 | 操作 | 预期结果 |
|----|------|------|---------|
| AC-29 | 用户 | 选中含外网图片且后端取不到 | 该图片替换为占位文本 `[图片加载失败：<原 URL>]`；不影响其它内容；不整批失败 |
| AC-30 | 用户 | 文件渲染进程异常 / 超时 | 返错误码 12064，`msg` 含 renderer 名便于排障；前端 toast「文件生成失败，请稍后重试」附简要 reason |
| AC-31 | 用户 | 请求 format 非 `docx` / `pdf` / `md` / `txt` | 返错误码 12063；前端不应允许触发，本码主要防直调 API |

## 3. 边界情况

- **多会话切换**：选择态下切到另一会话 → 自动退出选择态（清空选中 + 锚点 + 当前会话引用）。
- **页面刷新**：选择态不持久化，刷新即重置。
- **流式生成完成事件迟到**：以前端「复制」icon 的可见性条件为准（同一个 `isStreaming` flag）。
- **消息配对信息缺失**：legacy 数据可能不带配对字段；后端按「时间相邻、type 配对」兜底；前端联选找不到配对则只选当前一条，不报错。（具体配对策略与运行时坑见 design §3 决策1 / §5）
- **工作流 OUTPUT 节点多次输出**：单条答案可能含多条 output 事件；导出按顺序拼接成单段；非文本 output 走 AC-09 占位。
- **同名扫描冲突**：导入仅扫描目标层级（不递归）；扫描与入库间存在毫秒级 race，本期不做分布式锁；冲突时重命名重试最多 1 次，仍冲突返错误码 12069。
- **会话标题动态生成**：标题由后台 LLM 异步生成，可能未完成；为空或 `New Chat` 时统一用 `未命名会话`。
- **图片 URL 形态多样**：MinIO 内部地址 / 公网 http(s) / `data:` URL / 其它私有协议（跳过占位）。具体抓取策略与已知坑见 design §5。
- **临时文件清理**：每次请求在临时目录上下文中处理，响应完成自动清理，不持久化中间产物。
