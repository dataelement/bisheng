# Feature: F038-department-tree-lazy-load（部门树懒加载：按层加载 + 服务端搜索定位）

> **本文档定位 — 纯 What（需求口径，不随代码漂移）**
> spec 只回答 **做什么 / 验收标准 / 不做什么**。所有 How（架构决策、API 契约、数据模型、分层、组件/函数名、框架选型、性能实现）一律在 [design.md](./design.md) 与 [tasks.md](./tasks.md)。
>
> **前置步骤**：已完成 Spec Discovery（架构师提问 + 与用户对齐范围与搜索策略）。
> 触发动因：压测环境扩到 5 万部门后，部门树接口实测耗时约 6 秒、响应体约 14MB（一次性返回全部约 5 万节点）；授权页的部门主体列表更慢（约 69 秒，主因是按部门统计成员数）。

**关联 PRD**: 无独立 PRD（性能/体验优化型，源于 5 万部门压测实测）
**优先级**: P1（大组织规模下部门管理与授权选择的首屏可用性；不阻断核心业务，但规模放大后体验显著劣化）
**所属版本**: v2.6.0
**依赖**:
- F027-rebac-list-perf-optim（同属部门树端点族；已移除部门树节点 `member_count`，本特性进一步**推翻其 AC-16**，见 §2.6）
- F033-department-space-member-scope（授权部门列表的"按绑定部门子树收敛"范围）
- F026-channel-active-authorization（频道授权的部门主体列表与知识空间同源）
- F004 / F008（ReBAC core / resource-rebac-adaptation）、F011 / F012 / F013（多租户范围解析）

> **范围边界**
> - **本次纳入**：
>   - **部门管理树（platform 管理后台）**：整树一次性加载 → **按层懒加载**（首屏取根层；展开某节点时取其直接子节点；节点携带「是否有子节点」标识以决定展开箭头）。
>   - **服务端部门搜索**：取代现有"整树已加载后客户端递归过滤"；按部门名在用户可见范围内匹配，结果可直接在树中**展开定位**到命中部门。
>   - **定位到指定部门**：按部门 id 返回可在树中展开定位到该部门的结果，供选择器**回显深层的当前已选值**。
>   - **三层权限可见性**（系统管理员 / 部门管理员 / 租户管理员 / 非管理员）在懒加载、搜索、定位路径上**逐节点等价复刻**改造前的可见范围。
>   - **覆盖全部"部门树"消费面**：
>     - platform 的部门管理树，及其全部"选部门"入口（父部门选择、移动部门、角色范围、权限主体选择、部门知识空间映射、成员主属部门编辑、部门用户选择）。
>     - client（工作台）的**授权部门选择器**，出现在**知识空间共享**与**频道授权**两处。
>   - **授权部门列表移除「按部门成员数」统计**：该列表为每个部门附带的成员人数前端不展示、后端代价极大，予以移除（含其前端角标）。
>   - 迁移完成后**下线**旧的"整树一次性"部门树接口及其前端封装。
> - **本次明确排除**：
>   - **单父节点超宽**（某部门有上千直接子节点）的分页：本期单层全量返回；若后续要做必须遵循版本契约 INV-6（cursor-based，不返 `total`）。
>   - **整树服务端缓存**：懒加载后单次请求已足够小，本期不引入缓存层。
>   - 部门成员列表、成员搜索协议**不动**；本特性只改"部门树结构"的加载方式与上述成员数统计。
>   - 单部门详情的成员数保持现状（F027 AC-15 已定）。
>   - client 其它界面（除授权部门选择器外）不涉及。
> - **关键约束（边界，非 How）**：部门管理（platform）与授权管理（client）是**两套独立、不可混用的前端应用**——二者各自实现懒加载，不跨应用共用组件；其后端取数范围逻辑也不同（详见 design.md）。

## 1. 用户故事

**故事 A（大组织的部门/系统管理员）**：
作为 **管理后台里维护组织架构的管理员**，
我希望 **打开部门管理页时树能立即出现、点开某个部门时才加载它的下级**，
以便 **在几万部门规模下首屏不再等待约 6 秒、滚动与展开都顺滑**。

**故事 B（要在大树里找部门的管理员）**：
作为 **要快速定位某个部门的管理员**，
我希望 **输入部门名后树能直接展开并高亮到命中的部门**，
以便 **不必逐层手动展开，也不依赖"整棵树已在本地"才能搜。**

**故事 C（在弹窗里选部门的使用者）**：
作为 **创建/移动部门、配置权限、授权知识空间或频道时需要选部门的人**，
我希望 **选择器也按层加载、能搜索、并能回显我当前已选的那个（可能很深的）部门**，
以便 **选择器在大组织下同样秒开可用。**

**故事 D（管理员配置授权）**：
作为 **给知识空间/频道授权部门的管理员**，
我希望 **授权部门选择器在大组织下也能秒开，能搜到目标部门，已授权的部门有清晰标识**，
以便 **不被约 69 秒的加载卡住，且不会误重复授权。**

## 2. 验收标准

> AC-ID 在本特性内唯一，格式 `AC-NN`；tasks.md 测试任务通过 `覆盖 AC: AC-NN` 追溯。
> 错误码仅作"可观测对外行为"引用（如"返回权限错误"）；定义/归属在 design.md。
> 写法：复杂项用 EARS 句型；少数 UI 行为用表格。

### 2.1 按层懒加载（浏览）

- **AC-01** — WHEN 用户打开部门管理树, THE SYSTEM SHALL 仅返回其可见范围内的**根层节点**，不返回整棵树。
- **AC-02** — WHEN 用户展开某个节点, THE SYSTEM SHALL 按需返回**该节点的直接子节点**（一层），不附带更深层级。
- **AC-03** — THE SYSTEM SHALL 为每个返回的节点提供**「是否有子节点」标识**，使前端在不加载子节点的前提下能正确显示/隐藏展开箭头。
- **AC-04** — WHEN 用户折叠后再次展开同一节点, THE SYSTEM SHALL 不重复加载该节点子层（复用已加载结果），除非该子层因增删改被显式刷新。
- **AC-05** — WHEN 创建 / 移动 / 归档 / 恢复 / 重命名部门后, THE SYSTEM SHALL 使受影响父节点的子层刷新到最新（移动场景刷新原父与新父两处），无需整树重载。

### 2.2 服务端搜索与定位

- **AC-06** — WHEN 用户输入部门名关键词搜索, THE SYSTEM SHALL 在用户**可见范围内**按名称子串（大小写不敏感）匹配，返回**可在树中展开定位到命中部门**的结果，命中项带可识别标记。
- **AC-07** — IF 命中数量超过返回上限, THEN THE SYSTEM SHALL 返回截断标识，前端据此提示用户细化关键词。
- **AC-08** — IF 关键词为空或仅空白, THEN THE SYSTEM SHALL 返回空结果且不执行全表扫描式查询。
- **AC-09** — WHEN 用户清空搜索关键词, THE SYSTEM SHALL 回到懒加载浏览态，且此前已展开/已加载的浏览状态不丢失。
- **AC-10** — WHEN 选择器以一个已选部门打开（回显当前值）, THE SYSTEM SHALL 使该（可能很深的）部门在树中被展开并高亮，无需用户手动逐层展开。

### 2.3 权限范围逐层复刻

> 目标：懒加载/搜索/定位的可见性，与改造前**逐节点等价**。

- **AC-11** — WHERE 当前用户是系统管理员, THE SYSTEM SHALL 在根层与各层返回组织内对应的全部节点（与改造前整树可见集一致）。
- **AC-12** — WHERE 当前用户是部门管理员, THE SYSTEM SHALL 仅将其管辖部门及其子树纳入可见；根层返回其管辖部门本身（多个管辖范围有嵌套时只呈现最上层）。
- **AC-13** — WHERE 当前用户是租户管理员, THE SYSTEM SHALL 仅将其租户范围内的部门子树纳入可见；不得返回其范围之上的任何部门（搜索与定位的祖先回溯也止于该范围）。
- **AC-14** — IF 当前用户既非系统/部门/租户管理员（无任何管辖范围）, THEN THE SYSTEM SHALL 拒绝访问并返回权限错误（与改造前一致）。
- **AC-15** — IF 用户请求某个**不在其可见范围内**的节点的子层或定位到越权部门, THEN THE SYSTEM SHALL 返回权限错误，且响应不泄露越权节点的存在或名称。

### 2.4 platform 选部门入口与归档行为

> 本节针对 **platform 部门管理树及其选部门入口**；client 授权选择器见 §2.7。

| ID | 角色 | 操作 | 预期结果 |
|----|------|------|---------|
| AC-16 | 管理员 | 打开部门管理主树 | 树包含 active 与 archived 部门（与改造前一致），归档节点有可识别状态 |
| AC-17 | 选择部门的使用者 | 打开 platform 的父部门选择 / 移动目标 / 权限主体 / 知识空间映射等"选部门"入口 | 树按层懒加载、可搜索；列表中**不出现归档部门**（归档不可被选作父级/目标） |
| AC-18 | 选择部门的使用者 | 选择器的当前已选值恰为归档部门（回显场景） | 仍能展开并高亮回显该归档目标本身，但不将其作为可新选项 |

### 2.5 性能与下线（可观测目标）

> 性能为目标值（非硬性 AC，规模/环境相关）；接口下线用静态可验证方式。

- **AC-19**（目标）— 在 5 万部门规模下，打开树（根层）与展开任一节点（取子层）的单次接口耗时应**显著低于 1 秒**（对比改造前约 6 秒）；搜索单次请求同量级。
- **AC-20** — 迁移完成后对前端全仓静态扫描：不存在加载**整棵部门树**的调用方（旧整树接口与其前端封装已移除），所有部门树消费方走按层/搜索/定位接口。

### 2.6 授权部门列表移除成员数统计（推翻 F027 AC-16）

> **决策反转记录**：F027 AC-16 当时**特意保留**了授权页部门列表的成员数（理由"PRD 未明示移除"）。本特性据 5 万规模实测（成员数统计约占该接口耗时 96%）**推翻该决策**并移除。需在 F027 与版本契约留反转说明（SDD 防漂移）。

- **AC-21** — WHEN 调用授权页的部门主体列表（知识空间与频道授权两处）, THE SYSTEM SHALL 返回的每个部门节点**不含成员数字段**，且**不执行任何按部门统计成员数的查询**。
- **AC-22** — THE SYSTEM SHALL 在 client 授权部门选择器中不再展示部门名后的「(成员数)」角标。
- **AC-23**（目标）— 移除成员数后，该接口在 5 万部门规模下单次耗时从约 69 秒降到约 3 秒量级；后续叠加懒加载可进一步降到亚秒级。

### 2.7 client 授权部门选择器懒加载与选择语义（知识空间 / 频道）

> 范围：client 工作台中**知识空间共享**与**频道授权**两处的部门选择器（同一选择器）。它同样从"整树一次性 + 客户端过滤"改为按层懒加载 / 服务端搜索 / 定位，**且授予语义不变**。

- **AC-24** — WHEN 打开授权部门选择器, THE SYSTEM SHALL 仅加载根层并自动展开定位到本次已勾选的部门，不一次性加载整棵树；已授权部门**不主动全部展开**。
- **AC-25** — WHEN 在选择器中（浏览或搜索）遇到一个已授权部门, THE SYSTEM SHALL 将其渲染为禁用并标注「已授权」（显示而非隐藏，不可重复勾选）。
- **AC-26** — WHEN 在选择器中搜索, THE SYSTEM SHALL 返回可展开定位的命中结果，并按其「已勾选 / 已授权 / 隐式选中」状态呈现，与浏览态一致。
- **AC-27** — WHERE 某部门的祖先以「含子部门」被勾选, THE SYSTEM SHALL 将该部门呈现为隐式选中；「含子部门」对整子树**全有或全无**——隐式选中的子部门**不可被单独取消**（取消需改父级的「含子部门」开关）。
- **AC-28**（不变量）— THE SYSTEM SHALL 以「显式勾选的部门 + 是否含子部门」为授予真相提交；「已授权」集合来自资源已有授权而非已加载的部门树。**懒加载与否不得改变最终授予范围，也不得改变「已授权」判定**（见 §3 权限安全不变量）。

## 3. 边界情况

- **越权父节点 / 越权定位目标** → 返回权限错误，响应不含越权节点的 id/名称/存在性线索（AC-15）。
- **空 / 纯空白关键词** → 直接返回空结果，不触发全表扫描（AC-08）。
- **命中过多**（关键词匹配上万部门）→ 截断到上限并置截断标识，前端提示细化；不返回超大结果。
- **单父节点超宽**（直接子节点极多）→ 本期单层全量返回；若单层规模异常应可观测（记录告警），分页留作后续（遵循 INV-6）。
- **缓存陈旧**：本地增删改通过受影响父层刷新（AC-05）；他人并发改动造成的本地陈旧由刷新解决，属可接受。
- **归档父节点的子层**：管理主树允许展开归档节点查看子层；选部门入口默认不含归档（AC-17/AC-18）。
- **租户范围边界**：租户管理员可见性止于其范围；系统管理员可见全部（跨子租户语义沿用现状，本特性不改其机制）。
- **【权限安全不变量】授权选择器懒加载不得改变授权结果**：改造前加载整树**仅用于展示与导航**，没有任何权限决策依赖整树。授予范围由"显式勾选 + 是否含子部门"决定，"已授权"集合由资源已有授权决定。因此懒加载（不再加载整树）**不影响**"授予了谁 / 谁已授权 / 含子部门覆盖哪些"。**验收时须对懒加载前后、相同勾选下提交的授权做等价性核对。**
- 具体兜底（物化路径解析、授权服务不可用回退、超宽节点告警阈值、隐式选中的判定改造等）→ design.md §5。
