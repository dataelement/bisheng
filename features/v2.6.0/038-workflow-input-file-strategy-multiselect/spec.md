# Feature: 工作流输入节点文件处理策略多选

**关联 PRD**: [2.6 release · §工作流输入节点解析策略支持多选需求](https://dataelem.feishu.cn/wiki/NEtywYou0iJYfhka6RwcXkGAnpd)
**优先级**: P1
**所属版本**: v2.6.0
**依赖**: 无

> **范围边界**
> - **本次纳入**：
>   - 工作流输入节点「文件处理策略」由单选升级为「按文件类型（文档/图片）分组配置」。
>   - **对话框输入**：每个文件类型组内**单选**（解析内容 / 不解析）；变量按各组选择的并集联动展示。
>   - **表单输入**：策略**支持多选**（解析内容 / 存入临时知识库 可多选，不解析 互斥单选），至少选一个、不允许全空；配置项按所选并集融合展示。
>   - 数据结构升级：`file_parse_mode` 由字符串 → 对话框按类型 map、表单数组；历史单字符串向后兼容。
>   - 后端执行：按所选模式**集合累加**产出对应变量；对话框按文件类型分流套用各组策略。
> - **本次明确排除**：
>   - 历史工作流数据迁移（旧字符串运行时自动兼容，不批量改库）。
>   - 新增文件处理模式（仍是 extract_text / keep_raw / ingest_to_temp_kb 三种）。
>   - 对话框输入支持「同一文件类型组内多选」（PRD 明确组内单选）。

---

## 1. 用户故事

作为 **工作流搭建者**，
我希望 **上传文件时能对文档和图片分别指定处理方式、且需要时一种文件能同时产出多种结果**，
以便 **覆盖「文档解析成文本喂大模型、同时图片保留原图给多模态」「文件既解析又入临时知识库」这类单选满足不了的真实场景。**

---

## 2. 验收标准

### 2.1 对话框输入（组内单选 + 按类型分组）

| ID | 角色 | 操作 | 预期结果 |
|----|------|------|---------|
| AC-01 | 搭建者 | 上传文件类型选「全部类型」 | 分别展示「文档文件处理策略」与「图片文件处理策略」两个单选组 |
| AC-02 | 搭建者 | 上传文件类型仅「文档」/ 仅「图片」 | 仅展示对应一个分组 |
| AC-03 | 搭建者 | 任一可见组选「解析文件内容」 | 展示解析结果长度上限 + 变量 `dialog_files_content` |
| AC-04 | 搭建者 | 任一可见组选「不解析」 | 展示变量 `dialog_file_paths` |
| AC-05 | 搭建者 | 图片组选「不解析」且类型含图片 | 额外展示变量 `dialog_image_files` |

### 2.2 表单输入（可多选）

| ID | 角色 | 操作 | 预期结果 |
|----|------|------|---------|
| AC-06 | 搭建者 | 文件表单项选策略 | 「解析内容」「存入临时知识库」可同时勾选；「不解析」与其它互斥 |
| AC-07 | 搭建者 | 取消所有策略 | 阻止/回退，至少保留一个（默认「解析文件内容」） |
| AC-08 | 搭建者 | 勾选多个策略 | 按并集融合展示各自配置项（解析→长度上限+变量名；临时库→库名；不解析→路径变量名，含图片→图片变量名） |

### 2.3 数据结构与执行

- **AC-09** — THE SYSTEM SHALL 新保存的对话框策略以「按文件类型 map」结构存（仅含当前上传类型涉及的组）、表单策略以数组结构存。
- **AC-10** — WHERE 读到历史单字符串 `file_parse_mode`, THE SYSTEM SHALL 自动归一为单值/单元素，旧工作流执行结果不变。
- **AC-11** — WHEN 执行含多个策略, THE SYSTEM SHALL 一次解析、按所选模式集合分别产出对应变量（解析→content、不解析→path/image、入库→临时知识库），不重复解析。
- **AC-12** — WHERE 对话框上传「全部类型」且文档/图片选不同策略, THE SYSTEM SHALL 按文件扩展名分流：文档套文档组策略、图片套图片组策略。
- **AC-13** — 下游节点变量选择器 SHALL 只列出当前策略实际会产出的变量。

---

## 3. 边界情况

- 上传类型在「全部 ↔ 仅文档 ↔ 仅图片」切换时，隐藏组的策略与其变量应同步隐藏/清理，不残留无效变量。
- 表单输入「不解析」与「解析/入库」互斥：勾选「不解析」时其它取消，反之亦然。
- 解析文本总量仍受 `file_content_size`（默认 15000 字）上限约束，多策略下解析只做一次、产物复用。
- 历史值可能是字符串、也可能（升级后）是 map/数组 —— 读取侧必须三种都兼容。
